## 1. 확정 아키텍처 (3사 합의)

### 1-1. 모델 라우팅 — "직원 단위 → 작업 단위"로 전환 ⭐
가장 중요한 변경. 지금은 `staff.model = 'sonnet'|'haiku'` 한 칸. **한 직원 안에서도 작업 난이도가 다르다**(예: CS 분류는 싸게, 분쟁 답변은 비싸게).

**현행 가격(2026.6 기준, 답변 종합 — 배포 전 재확인):**

| 모델 | 입력/출력($/1M) | Vision | 실시간검색 | 용도 |
|---|---|---|---|---|
| Claude Sonnet 4.6 | $3 / $15 | ✅ | ❌ | 브랜드톤·한국어·구조화 (기본 워크호스) |
| Claude Opus 4.8 | $5 / $25 | ✅ | ❌ | 대표상품 롱폼·고난도만 |
| Claude Haiku 4.5 | $1 / $5 | ✅ | ❌ | 분류·요약·저가 고속 |
| GPT-5.4 | $2.50 / $15 | ✅ | 툴 | 카피 변형 대량·후킹 (가성비) |
| GPT-5.5 | $5 / $30 | ✅ | 툴 | 고난도 카피 |
| Gemini 3.1 Pro | $2 / $12 | ✅ | 그라운딩 | 제품사진 Vision·가성비 |
| Perplexity Sonar Pro | $3 / $15 | ✅ | ✅(인용내장) | 시장조사·경쟁사·트렌드 수집 |

> ⚠️ 설계서의 `GPT-4o`·`claude-sonnet-4-20250514`는 **구세대** → 위 라인업으로 교체.
> DB: `staff_model_routes(staff_type, task_type, provider, model_id, fallback_*)` 신설 (작업별 매핑).

### 1-2. 프롬프트 5겹 — 구조 유지 + 캐싱 + ④ 슬림화 ⭐
구조는 옳다(3사 동의). 단 **두 가지를 고친다**:
1. **①회사브레인+②SOP+③직원프롬프트 = 정적 프리픽스 → 프롬프트 캐싱**(캐시 읽기 = 입력가의 0.10×, **90% 절감**). ⑤작업지시는 **맨 뒤**(캐시 보존).
2. **④동료 산출물 = 전체 리포트 주입 금지.** "인사이트 공용 보드에서 내 역할 관련 항목만 구조화 선별 주입"(직원당 ≤1,500토큰).

```
[정적·캐싱]  ① 회사 브레인(brand_contexts)  ┐
            ② 타입 베이스 SOP(코드)         ├─ 5분/1시간 TTL 캐시
            ③ 직원 개별 프롬프트            ┘
[동적]      ④ 관련 인사이트(보드에서 선별, 요약)
            ⑤ 작업 지시(수동입력 or 일과) ← 맨 뒤
            ⑥ 출력 스키마(JSON)
```
> cron이 5~15분 간격이면 캐시 미스 위험 → **일과를 묶어 연속 호출**하거나 1시간 TTL.

### 1-3. [액션] 파싱 — 마크다운 → JSON ⭐
현재 `일정: ... | 제목` 마크다운 파싱은 **깨지기 쉽다**(GPT·클로드 둘 다 지적). **JSON/tool schema**로 받는다:
```json
{ "report": { "title": "...", "summary": "...", "body": "..." },
  "actions": {
    "tasks":     [{ "title": "...", "priority": "high", "due_at": "2026-06-16T09:00+09:00" }],
    "schedules": [{ "title": "...", "at": "..." }],
    "insights":  [{ "title": "...", "body": "...", "tags": ["원목도마","관리쉬움"], "confidence": 0.8 }]
  } }
```

### 1-4. 저장 구조 — "리포트 한 곳" → 역할별 분리 ⭐
`daily_reports`에 다 쌓으면 "광고 카피만 비교", "실패 재시도"가 불가. 분리:

| 테이블 | 역할 |
|---|---|
| `staff_runs` | **실행 원장** — trigger(auto/manual), status, provider, model, tokens, cost, error |
| `staff_outputs` | **산출물** — output_kind, content_md, content_json, source_run_id |
| `staff_output_actions` | [액션] — type, status(**suggested→approved→done/dismissed**), payload |
| `content_assets` | **재사용 자산** — ad_copy/sns_post/detail_section/image, status(draft/approved/used) |
| `insights` | **사내 지식 DB** — +source_url, confidence, valid_until, related_products, used_by |
| `brand_contexts` | **회사 헌법** — 정체성/톤/USP/금지어 (workspace 1:1, +version) |
| `compliance_rules` | **카테고리별 금지표현** — category, rule, severity, examples_bad/good |
| `daily_reports` | 하루 요약 **집계 뷰**로 격하 (원장 아님) |

### 1-5. 협업 — 인사이트 보드 + 가벼운 의존 힌트
전체 리포트 주입(노이즈 폭탄) ❌. **보드에 쓰기 → 다음 실행 때 관련 직원이 읽기**(비동기 이벤트). 코드 상수로 우선순위만:
```ts
const staffContextMap = {
  ad:          ['monitor', 'sourcing', 'analyst'],
  sns:         ['monitor', 'visual', 'ad'],
  detail_page: ['sourcing', 'visual', 'cs'],
  sourcing:    ['monitor', 'analyst'],
  monitor:     ['sourcing'],
  analyst:     ['ad', 'sns'],
  visual:      ['detail_page', 'sns'],
  cs:          ['detail_page'],
};
// 실제 내용은 보드에서 "관련 태그 최신 N건 요약"만 동적 선별
```

### 1-6. 자동+수동 UX — 트리거 분리, 결과 통합
- 직원 상세 = **상단 "오늘의 일과(자동)" + "지금 시키기(수동)" 입력창** → 결과는 **단일 활동 타임라인**에 `🕒자동`/`👤수동` 배지 + 모델·비용 메타.
- 수동 실행 후 **"이 패턴을 일과로 저장" CTA**(반복 작업 자동화 승격).
- **민감 액션(외부 게시·광고비·CS 발송)은 초안→승인→실행 게이트(HITL)** + 감사로그/롤백.

## 4. ⭐ 직원별 사용 시뮬레이션

> 각 시뮬레이션 = `[입력] → [프롬프트 조립] → [모델] → [출력] → [메뉴 반영]` 전 과정.

### 🔍 소싱 기획자 — "이 아이템 팔까?"

**[수동] 사장이 직접 아이템 분석 시키기**
```
사장: (직원 상세 "지금 시키기"에 입력 + 이미지 업로드)
      📎 teak_board.jpg
      "이거 티크 원목 도마인데 39,000원에 소싱 가능. 우리가 팔만한지 봐줘."
      [▶ 실행]
```
**내부 흐름 (작업 단위 라우팅):**
1. STEP1 `Gemini 3.1 Pro Vision` → 이미지 구조화
   `{ product:"티크 원목 도마", material:"wood", style:["natural","kitchen","premium"], uncertain:["목재 종류 확정 불가"] }`
2. STEP2 `Perplexity Sonar Pro` → "티크 원목 도마 2030 신혼 타겟 2026 한국 가격대·경쟁사·리뷰키워드" (**인용 URL 저장**)
3. STEP3 `Claude Sonnet 4.6` → 회사브레인(시목 톤·USP) + 보드의 monitor 인사이트("원목도마 친환경 키워드 급상승") 결합 → **소싱 브리프**

**[출력] `sourcing_brief`:**
```
# 티크 도마 — 추천 ✅ (근거 3줄)
· 시장: 3.5~5만원대 형성, 39,000원은 중상 포지션 OK
· 구매장벽 1위 "관리 어려움" → 시목 USP '천연오일 마감'으로 정면 돌파
· 경쟁 키워드 "통원목/친환경" 상승세, 진입 적기
[예상마진] 원가 18,000 / 판매 39,000 / 마진율 54%
[경쟁사] A몰 42,000 · B몰 35,000(세트) · C몰 38,000
[리스크] 목재종류 이미지 확정불가 → 입고 시 검증 필요
[출처] https://... https://...   ← 사장 검증용
```
**[메뉴 반영]** 액션 → 인사이트 보드 "원목도마 관리쉬움=USP"(자동), 할일 "티크도마 샘플 입고 검증"(`suggested`).

**[자동] 매일 09:00 일과**
→ "카테고리 트렌드 점검" 라벨이 ⑤에 주입 → Perplexity로 원목/주방 카테고리 급상승 키워드 수집 → 보드에 인사이트 적재(다음날 광고/SNS 직원이 읽음).

```
[전략 브리핑] 타겟: 신혼·자취 / USP: 통원목·천연오일 / 금지어: "평생보장","100%무독성"(근거필요)

┌ ... 6섹션 ...
[금지표현 검사] ✅ 통과 (효능 단정 없음)
[HTML 복사] [PC/모바일 미리보기]
```

**[메뉴 반영]** → `content_assets`에 `detail_section` draft 저장(나중에 "성과 좋았던 카피로 다시" 가능). image_brief 슬롯은 비주얼 디렉터와 연동.

### 💬 CS 응대 — "문의 쌓였어, 답변 초안"

**[자동] 실시간 문의 1차 응답 + 매일 18:00 요약**
```
고객 문의 유입 → 1) Haiku 분류(반품/배송/불만/단순문의 + 감정)
              → 2) Sonnet 답변 초안(공감→해결, 3~5문장, 정책 내 수용적)
              → 3) 분쟁/환불 건은 [사람 승인 대기]로 플래그
```
**[출력] `ticket_list`:**
```
🔴 [불만·긴급] "도마에 곰팡이가..." → 초안: "불편을 드려 정말 죄송합니다. 원목은 첫 사용 전
   오일 관리가 필요한데 안내가 부족했네요. 교환 도와드리고 관리법 영상 보내드릴게요."
   [복사][FAQ에 반영][발송은 승인 후]  ← 민감건 HITL
```

## 5. 우선순위 로드맵

**P0 — 배포 전 필수 (리스크·비용 직결)**
1. 모델 버전 현행화 + `staff_model_routes`(작업 단위 라우팅)
2. 프롬프트 캐싱(①②③ 정적 프리픽스) + ⑤ 맨 뒤 배치
3. [액션] JSON 스키마화 (마크다운 파싱 폐기)
4. HITL 승인 게이트 + 액션 `suggested→approved→done` 상태값 + 감사로그
5. 건강식품 `compliance_rules` 가드레일

**P1 — 출시 직후 (제품성↑)**
6. `staff_runs`/`staff_outputs`/`content_assets` 저장 분리
7. ④계층 → 인사이트 공용 보드(구조화 선별 주입, ≤1,500토큰/직원)
8. 소싱·모니터링 2단 파이프라인(Vision→Perplexity→Claude, 인용 저장)
9. 운영 매니저 직원 추가
10. 비용 관측 대시보드 + 예산 상한/알림

**P2 — 확장기 (진짜 회사)**
11. 이미지 생성 API 직접 연동(Imagen 4 Fast/Nano Banana/gpt-image-1)
12. 운영(주문/재고/정산) — 네이버 커머스API/쿠팡 제약 반영
13. 품질 평가 루프(사장 채택/반려 → 프롬프트 개선)
14. 커지는 데이터(카탈로그·CS히스토리)에만 RAG(pgvector) — 회사 브레인엔 금지
