# 🖼️ 전체 구조 — 그림으로 이해하기

> 머릿속에 그림이 안 그려질 때 보는 문서. 화면 목업 + 하루 시나리오로 "실제로 어떻게 돌아가는지"를 봄.
> **두 트랙이 한 앱 안에서 같이 돈다:** ①24시간 AI 오피스(AI가 자동으로 일함) + ②공유 워크스페이스(남편이랑 같이 봄).
> **작성일:** 2026-06-12

## 0. 큰 그림 — 한 앱, 두 능력, 한 접점

```
                        ┌──────────────────  Sol AI Office (지금 그 앱)  ──────────────────┐
                        │                                                                  │
   ① 24시간 AI 오피스    │   AI 직원들(서치·플래니·마키·데비)이 매일 새벽 자동으로 일함        │
   (자동으로 일함)       │   → 경쟁사·시장·고객·소구점·상세기획·인사이트 → "오늘의 리포트+할일"  │
                        │                              │                                    │
                        │                      ┌───────┴────────┐                          │
   ★ 둘이 만나는 곳      │                      │   시목(공동)    │  ← 부부 공동 브랜드        │
                        │                      └───────┬────────┘                          │
                        │                              │                                    │
   ② 공유 워크스페이스   │   나 ─── 공유 토글 ───►  같은 화면  ◄─── 공유 토글 ─── 남편         │
   (같이 봄)            │   개인 건 나만, 시목 건 둘이 같이 보고 서로 할일 추천                 │
                        └──────────────────────────────────────────────────────────────────┘
```

- **운명랩·쏠닝포인트** = 나 혼자(개인). **시목** = 부부 공동.
- 시목의 AI 리포트는 **자동으로 둘 다에게** 보임. 개인 브랜드는 나만 봄.

## 1. "공간" 구조 — 폴더처럼 생각하면 쉬움

워크스페이스 = **방(누가 들어오나)**, 프로젝트 = **그 방 안의 서랍(무슨 일이냐)**.

```
🗂️ 개인 워크스페이스 "내 오피스"   (나만 들어옴)
     ├─ 📁 운명랩        ← 프로젝트(브랜드)
     ├─ 📁 쏠닝포인트     ← 프로젝트(브랜드)
     └─ 📁 개인 일정/기록/독서…

🗂️ 팀 워크스페이스 "시목"          (나 + 남편 들어옴)
     └─ 📁 시목          ← 프로젝트(브랜드) + AI 직원이 매일 일하는 곳
          ├─ 일정·할일·인사이트·기록 (둘이 공유)
          └─ 🤖 오늘의 AI 리포트 (자동)
```

> 핵심: **"어느 방에 넣었느냐"가 공유를 정함.** 시목 방에 넣은 할일은 기본 공유, 내 오피스에 넣은 건 나만. (단, 일기는 시목 방에 있어도 기본 비공개)

## 2. 화면이 실제로 이렇게 보임 (목업)

### 2-1. 맨 위 "공간 전환" 바 (항상 보임)
```
┌──────────────────────────────────────────────────────────────┐
│  Sol AI Office      [ 🌐 전체 ] [ 👤 내 오피스 ] [ 🪵 시목 ]   👤 │
└──────────────────────────────────────────────────────────────┘
        클릭하는 칩에 따라 아래 내용이 바뀜 (다중 선택도 가능)
```

### 2-2. "🌐 전체" — 흩어짐 방지용 통합 뷰 (정신없음 해결!)
> 내 모든 공간의 일정·할일을 **한 화면에** 모아줌. 색 점으로 어디 건지 구분.
```
┌ 오늘 할일 (전체) ───────────────────────────────────┐
│ 🟣 운명랩  인스타 릴스 촬영                    [ ]   │
│ 🟢 쏠닝포인트  상세페이지 후기 수집             [ ]   │
│ 🪵 시목  원목 오일 재입고 발주        (남편 담당) [ ] │
│ 🪵 시목  도마 상세페이지 검토            (내 할일) [ ] │
│ ⚪ 개인  병원 예약                            [ ]   │
└─────────────────────────────────────────────────────┘
   🟣🟢⚪ = 내 개인 공간   🪵 = 시목(공유, 남편도 같은 걸 봄)
```

### 2-3. "🪵 시목" — 공유 공간만 보기
```
┌ 시목 ───────────────────────────────  나 · 남편 함께 보는 중 │
│ 🤖 오늘의 AI 리포트  "경쟁사 3곳 가격 인하…"      [ 펼치기 ] │
│ ─────────────────────────────────────────────────────────  │
│ 할일   도마 상세페이지 검토         (내 할일)        [ ]     │
│ 할일   원목 오일 재입고 발주        (남편 담당)      [ ]     │
│ 인사이트  "선물 수요 12월 급증" — 내가 공유함                 │
└────────────────────────────────────────────────────────────┘
```

### 2-4. 할일 만들 때 — 토글 하나
```
┌ 할일 추가 (지금 위치: 🪵 시목) ──────────┐
│ 제목: [ 도마 상세페이지 검토        ]     │
│ 마감: [ 6/15 ]                           │
│ [✓] 시목에 공유      ← 시목 방이라 기본 ON │
│   └ 담당:  ( 나 ●)  ( 남편 ○)            │
└──────────────────────────────────────────┘
   "남편" 선택 + 저장 → 남편 할일로 즉시 생성 + 남편한테 푸시 📲
```

## 3. 하루 시나리오 (이게 핵심 — 실제로 이렇게 흘러감)

```
🌅 새벽 5시 — 내가 자는 동안
   24시간 AI 오피스가 자동으로 켜짐.
   서치 → 경쟁사 3곳 가격 확인   |  플래니 → 시목 고객 분석
   마키 → 마케팅 소구점 정리      |  데비 → 도마 상세페이지 초안
   → "시목 오늘의 리포트" 1장으로 묶임 + "오늘 할일 5개" 자동 생성
   → 시목은 공유 프로젝트라 → 나·남편 둘 다에게 저장 + 푸시 📲

☀️ 아침 8시 — 나
   폰 알림 "시목 오늘의 오피스 리포트 도착 📋"
   앱 열기 → [🪵 시목] → 리포트 읽음.
   할일 5개 중 "원목 오일 재입고 발주"는 남편이 잘 함
   → 그 할일 담당을 (남편)으로 → 즉시 남편 할일로 감 + 남편 푸시 📲

🏢 낮 — 남편
   앱 열기 → [🌐 전체] → 자기 받은 할일 "원목 오일 재입고 발주" 보임
   발주 완료 → 체크 ✅  (나는 그 변화를 시목 화면에서 같이 봄)

🌙 저녁 — 나
   [👤 내 오피스] 로 전환 → 운명랩·쏠닝포인트·개인 일기 정리
   (이건 남편 안 보임. 토글 안 켜면 100% 나만.)
```

## 4. ① 24시간 AI 오피스가 도는 모습 (자동 파이프라인)

```
[매일 새벽 자동 실행 — 내가 아무것도 안 해도]

  STEP1 경쟁사  ─┐
  STEP2 시장     │  AI 직원들이 순서대로 일함
  STEP3 고객     │  (각 단계 결과가 다음 단계 재료로 흘러감)
  STEP4 소구점   │
  STEP5 상세기획 │
  STEP6 인사이트 ─┘
        │
        ▼
  STEP7  📋 "오늘의 리포트" 저장  +  ✅ "오늘 할일" 자동 생성  +  📲 푸시
        │
        └─ 시목(공유) → 부부 공통 / 운명랩·쏠닝(개인) → 나만
```
> 핵심: 이미 우리 앱엔 **새벽에 자동으로 도는 장치(morning-briefing 등)가 있음.** 거기에 "AI가 리포트 쓰는 단계"만 얹는 것.

## 6. ⚠️ 가장 중요한 안전장치 (남편 초대 전에 반드시)

```
지금 모디(AI)는 "내 모든 정보"를 보고 답함.
그대로 두면 → 남편이 시목 방에서 모디한테 물었을 때
            내 운명랩 매출/KPI가 답변에 새어나올 수 있음 😱

해결: AI가 보는 정보를 "지금 이 방(워크스페이스) + 공유된 것"으로 제한.
      → 남편 초대(이메일)는 이 작업 끝낸 다음에.
```

## 7. 빌드 A부터 — 단계마다 화면에서 뭐가 바뀌나

| 빌드 | 만드는 것 | 사장님 눈에 보이는 변화 |
|---|---|---|
| 🟦 **A 기반** | 워크스페이스 테이블·백필·RLS (내부 공사) | **겉보기 그대로** (안 깨졌는지 확인용. 안전!) |
| 🟩 **B 안전장치** | AI 컨텍스트 스코핑 + 푸시 멤버 전원 | (내부) 남편 초대해도 정보 안 새게 |
| 🟨 **C 공개기능** | 상단 공간 전환바 + 공유토글 + 받은할일 + 🌐통합뷰 + 이메일 초대 | **여기서 처음 "공유"가 눈에 보임** + 남편 초대 가능 |
| 🟪 **D 합류** | 시목 AI 리포트를 부부 공통 위젯으로 | 대시보드에 "시목 오늘의 리포트" 등장 |

> 🟦 A는 "겉은 그대로, 속만 바꾸기"라 가장 안전 → 배포해서 멀쩡한지 확인하고 다음으로.

## 8. 역할 분담 — 누가 뭘 함

| | 사장님이 함 | Claude가 함 |
|---|---|---|
| **공유 워크스페이스** | 시목 팀 워크스페이스 "생성", 남편 이메일 초대 | 테이블·RLS·전환바·공유토글·통합뷰 **기능 구현** |
| **24h AI 오피스** | 시목 브랜드 컨텍스트 입력, 경쟁사 자료 붙여넣기 | 자동 파이프라인·AI 프롬프트·리포트 화면 **구현** |
| **상세페이지봇** | (선택) 더미 #2~5 더 돌리기 | 역설계 결과를 STEP5에 탑재 |

> 즉, **틀·기능은 Claude가 만들고, 시목의 실제 내용·초대는 사장님이 채움.** 두 트랙(공유 + 24h)이 같이 굴러감.

# 맞는지
시뮬레이터에서 확인해볼 만한 시나리오 세 가지만 짚으면:

남편 + 통합 — Sol의 운명랩·쏠닝 항목은 점선 카드로 "행 자체 미반환"이 됩니다. 멤버가 아닌 워크스페이스는 숨겨지는 게 아니라 DB 쿼리 결과에 아예 안 들어오는 것, 이게 RLS의 동작 방식입니다.
남편 + 우리(팀) — "시목 회고 일기"는 같은 팀 워크스페이스인데도 잠겨 있죠. journals만 비공개 default라는 §3 규칙이 여기서 작동합니다. 반대로 "시목 매출 메모"를 Sol 시점으로 보면 같은 이유로 잠깁니다.
Sol + 우리(팀) — "시목 원목 자재 발주"는 남편이 만들었지만 assignee=Sol이라 "받은 할일" 배지와 함께 보입니다. is_shared와 무관하게 담당자는 항상 본다는 RLS 절이 이것입니다.

그리고 하단 푸터의 "AI 컨텍스트 주입 대상" 숫자가 §6의 핵심을 보여줍니다 — 화면(RLS)은 자동으로 걸러지지만, 모디의 시스템 프롬프트는 service role로 조립되기 때문에 저 숫자만큼만 주입되도록 코드에서 직접 걸러야 남편이 시목 방에서 질문했을 때 운명랩 KPI가 새지 않습니다. 빌드 순서에서 B(스코핑)를 C(초대)보다 먼저 두신 이유가 정확히 이 그림이에요.
