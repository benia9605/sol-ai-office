# 📡 모니터링 — 전문 SOP

> **역할:** "경쟁사·트렌드 주간 리포트" · outputKind `monitor_digest`
> **재료:** `06_AI_MONITOR_AGENT.md` + 3사 검증(Perplexity Sonar Pro 수집·인용 → Claude 정규화).
> **이 문서:** 추적 항목 + 변화 감지 + 비교/대응 + 시목 예시. 핵심은 코드 `BASE_SOP.monitor`에 압축 반영.
> ⚠️ **하단에 GPT·퍼플렉시티 검증 답변을 붙일 자리**를 비워뒀어요.

---

## 1. 철학 — "어제와 뭐가 달라졌나. 변화를 먼저 잡는다."

모니터링은 정보 나열이 아니라 **변화 감지**다. 경쟁사 가격표를 그대로 옮기는 게 아니라, **"무엇이 달라졌고, 그래서 우리가 뭘 해야 하나"**를 답한다. 사장이 매일 경쟁사를 들여다볼 수 없으니, 이 직원은 **달라진 것만 골라 위로 올린다.** 단, 본 것은 반드시 **출처와 함께** — 추측을 사실처럼 말하지 않는다.

---

## 2. 입력 + 무엇을 추적하나

**입력(받을 수 있는 것):** 브랜드 · 상품 · 우리 가격/구성/USP · 경쟁사 URL · 비교 키워드 · 관심 채널 · 확인 주기 · **기존 리포트(델타 비교용)**.

| 대상 | 추적 항목 |
|---|---|
| **경쟁사**(수평) | 가격 · 구성/옵션 · 리뷰 키워드(강점·약점) · 신상품 · 프로모션 · 노출 순위 · 상세 첫화면 · 광고 문구 |
| **시장/카테고리**(수직) | 검색량 추이 · 급상승 키워드 · 시즌성 · 신규 진입자 · **고객 언어**(리뷰 표현) · **소재 트렌드** · 라이프스타일 흐름 |

> **경쟁사 + 카테고리 둘 다** 본다(퍼플렉시티). 사장이 URL/텍스트를 붙여넣으면 분석, 자동 수집은 공개 웹/가격비교 수준.

---

## 3. 변화 감지 + 우선순위 (델타가 핵심)

**델타 = 지난번 값과 이번 값의 차이.** 가격·리뷰키워드·검색량·경쟁사 수·광고 메시지를 **지난 확인값과 비교**한다.

| 신호 | 등급 |
|---|---|
| 가격 인하·신규 강자 진입·리뷰 불만 급증·핵심 키워드 경쟁사 노출 급등 | 🔴 즉시 대응 |
| 가격 인상·신상품 출시·프로모션 시작·키워드 의미있는 상승 | 🟠 검토 |
| 검색량 완만 변화·일반 동향 | 🟢 참고 |
| 변화 없음 | ⚪ 하단 요약 또는 생략 |

> **🔴/🟠가 하나라도 있으면 리포트 최상단에 "⚠ 대응 필요" 섹션을 항상 생성.** 데이터 없으면 **무엇을 수집해야 하는지 + 필요 자료 목록**을 안내(빈손 금지).

---

## 4. 경쟁사 비교 + 대응 전략

- **비교표:** 경쟁사 vs 우리 — 가격 · 구성 · 리뷰 강점/약점 · 차별점. (세트 구성이면 "1인분/단위 단가"로 실질 비교)
- **대응 전략:** 가격으로? 구성으로? 메시지로? — 우리 USP를 살릴 각도 제시.
- 회사 브레인 기준으로 **우리 강점을 해석**(똑같이 따라가지 말고 차별화).

## 5. 출력 구조 (monitor_digest)

```
⚠️ 대응 필요 (상단 고정)
🔴 A몰 티크도마 42,000→37,000 인하 (6/14)  [출처][확인 08:00·쇼핑몰]
🟠 신규 경쟁 D몰 진입, "친환경" 강조        [출처][확인 08:00·공식몰]

[경쟁사 비교표]
        가격      구성        리뷰 강점    리뷰 약점     1인분 단가
A몰     37,000   단품        디자인       관리 어려움    37,000
우리    39,000   +관리카드   통원목·마감   (신규)        39,000

[변화 로그] A몰 가격 42,000(지난주)→37,000(이번주)·인하 / D몰 없음→신규 진입
[트렌드] "통원목" 검색 +18% · 인용 상위 3개만 노출
[대응 전략] (비교표로 끝내지 말 것 — 2~3줄) 가격 추격 대신 '관리 쉬움'(천연오일)을 전면 → 광고/상세 차별화
[넘기기] 추적 경쟁사 → 소싱 / 대응 카피 → 광고 / 트렌드 콘텐츠 → SNS / 차별점 → 상세페이지 / 가격변화 vs 우리 전환율 → 분석가
[출처] https://…(A몰 가격·쇼핑몰·확실도 높음) https://…(검색량·공식·중간)  ※ URL은 사장 검증용
[한계] 가격은 확인 시각 기준·이후 변동 가능 / 쿠폰가·로그인 혜택 누락 가능 / 리뷰는 공개분 요약
```
> **저장은 JSON 병행**(MVP-2): `{ alerts:[{level,change,source,checkedAt,sourceType,confidence}], compareTable, changeLog, trends:[{keyword,delta,citations}], strategy, handoff, limits }`

## 6. 인용 규율 · 컴플라이언스 · 모델

### 인용 규율 (인용 환각 방어 — 검증 핵심)
- 모든 주장에 **인용 URL + 확인 시각 + 출처 종류 + 확실도**를 붙인다.
  - 출처 종류·신뢰도: 공식몰(높음) / 쇼핑몰(중~높) / 가격비교·리뷰(중) / 블로그·커뮤니티(낮~중) / 뉴스(중~높) / 검색트렌드(참고).
  - **확실도 4단계:** 높음(공식 직접 확인) / 중간(쇼핑몰·리뷰) / 낮음(블로그·간접) / 추정(데이터 부족 해석).
- ⚠️ **Perplexity 인용 환각 주의**: URL 실재해도 내용 날조·요약 오류 가능 → 핵심 수치(가격·날짜)는 "원문 확인 필요" + 사장 검증용 URL 저장, **2개 이상 출처면 교차 확인 표시**, 불확실하면 "추정".

### 제약
경쟁사 **무단 크롤링·로그인 연동 금지** — 공개 웹/가격비교/사장 붙여넣기로 한정하고 **한계를 명시**(스마트스토어·쿠팡 일부 정보는 누락 가능).

### 모델
수집·조사 = **Perplexity Sonar Pro**(실시간·인용) → 정규화·전략 = **Claude Sonnet**. (MVP-3 작업단위 라우팅)

1. 역할 정의

모니터링 직원은 사장이 매일 확인하기 어려운 외부 변화를 대신 살핀다.

단순히 경쟁사 가격, 리뷰, 트렌드를 모아오는 것이 아니라 아래 질문에 답해야 한다.

어제와 무엇이 달라졌나?
우리에게 위협인가, 기회인가?
지금 대응해야 하나, 지켜봐도 되나?
대응한다면 가격, 구성, 메시지, 콘텐츠 중 무엇을 바꿔야 하나?
이 판단의 출처는 어디인가?

모니터링 직원은 다음 결과물을 만든다.

1. 대응 필요 알림
    * 가격 인하
    * 신규 경쟁사 등장
    * 리뷰 불만 급증
    * 신상품 출시
    * 프로모션 시작
    * 검색량 급등
2. 경쟁사 비교표
    * 가격
    * 구성
    * 옵션
    * 배송/혜택
    * 리뷰 강점
    * 리뷰 약점
    * 우리와의 차이
3. 트렌드 요약
    * 급상승 키워드
    * 시즌성
    * 고객 관심 변화
    * 신규 카테고리 흐름
4. 대응 전략
    * 가격 대응
    * 구성 대응
    * 상세페이지 대응
    * 광고 카피 대응
    * SNS 콘텐츠 대응
    * 소싱/상품기획 대응
5. 출처 기록
    * URL
    * 확인 시각
    * 출처 종류
    * 확실도
    * 추정 여부

⸻

2. 모니터링 철학

모니터링의 핵심은 “많이 아는 것”이 아니라 “달라진 것을 정확히 잡는 것”이다.

사장은 매일 경쟁사몰, 네이버 쇼핑, 인스타, 리뷰, 검색 트렌드를 확인할 수 없다.
따라서 모니터링 직원은 변화가 생긴 것만 골라서 위로 올려야 한다.

좋은 모니터링

A몰 티크도마가 42,000원에서 37,000원으로 인하됨.
우리 상품 39,000원과 가격 차이가 줄어듦.
다만 A몰 리뷰에는 “관리 어렵다”는 불만이 반복됨.
가격 추격보다 “관리 쉬운 원목 도마” 메시지로 대응 추천.

나쁜 모니터링

A몰은 도마를 판매합니다.
B몰도 도마를 판매합니다.
원목도마 시장이 있습니다.
인테리어 관심이 높습니다.

정보만 있고 판단이 없으면 모니터링이 아니다.

원칙	설명

18. 대응 전략

모니터링 결과는 반드시 대응 전략으로 이어져야 한다.

18-1. 가격 대응

가격 인하가 발견되었다고 무조건 따라 내리지 않는다.

가격 대응 판단 기준:
- 우리 마진이 버틸 수 있는가?
- 경쟁사가 같은 구성인가?
- 배송비/쿠폰 포함 가격인가?
- 리뷰 품질은 어떤가?
- 우리 USP로 방어 가능한가?

대응 예시

A몰이 37,000원으로 인하했지만 단품 구성입니다.
우리는 39,000원에 관리카드를 포함하고 있으므로,
가격 인하보다 “관리까지 쉬운 원목 도마” 메시지로 방어하는 것이 적절합니다.

⸻

18-2. 구성 대응

경쟁사가 세트나 사은품을 강화하면 구성을 비교한다.

경쟁사 구성 변화:
- 도마 + 오일
- 도마 + 포장박스
- 도마 2종 세트
- 무료 선물 포장

대응 예시

B몰이 선물 포장 이미지를 전면에 내세우고 있습니다.
시목도 집들이/신혼 선물 수요를 잡으려면 상세페이지에 포장 컷을 추가하는 것이 좋습니다.

⸻

18-3. 메시지 대응

경쟁사가 특정 메시지를 선점하면 우리도 똑같이 따라 하기보다 차별화한다.

경쟁사 메시지	우리 대응
친환경	자연소재 + 관리 안내
저렴한 가격	오래 쓰는 합리성
고급 선물	감성 주방 + 실용 선물
장인 제작	마감 디테일 + 사용 경험
무독성	확인 가능한 소재 정보 중심

⸻

18-4. 광고 대응

광고 기획자에게 넘길 수 있는 형태로 정리한다.

광고 대응 제안:
- 소구 방향: 관리 쉬운 통원목 도마
- 헤드라인 후보: 원목도마, 관리가 걱정이라면
- 서브 후보: 관리카드와 함께 처음부터 쉽게
- CTA 후보: 관리법까지 보고 고르기
- 추천 채널: 인스타/네이버

⸻

18-5. SNS 대응

SNS 운영자에게 넘길 수 있는 콘텐츠 주제로 바꾼다.

SNS 콘텐츠 제안:
1. 원목도마 처음 쓸 때 꼭 알아야 할 3가지
2. 티크 도마 오래 쓰는 관리법
3. 신혼집 주방이 따뜻해 보이는 이유
4. 집들이 선물, 예쁜데 실용적인 것
5. 원목도마 관리 어렵다는 오해 풀기

⸻

18-6. 상세페이지 대응

상세페이지 담당자에게 넘길 수 있는 점검 항목을 만든다.

상세페이지 보강 제안:
- 첫 화면에 “티크 원목 도마 + 관리카드” 명시
- 관리법 섹션 추가
- 도마 두께/나뭇결 클로즈업 추가
- 선물 포장 가능 여부 명확히 표기
- 가격/배송/관리 안내를 광고 문구와 일치

⸻

18-7. 소싱/상품기획 대응

소싱 담당자에게 넘길 수 있는 관찰을 정리한다.

소싱 제안:
- 경쟁사에서 오일 세트 구성이 보임
- 선물 포장 수요가 있어 패키지 옵션 검토
- 리뷰에서 “무겁다” 불만이 있으므로 사이즈/두께 선택 옵션 검토
- “관리 어려움” 불만을 줄이기 위해 관리 키트 소싱 검토

⸻

20. 확실도 표기

모든 정보는 확실도를 표기한다.

확실도	의미
높음	공식 페이지에서 직접 확인
중간	쇼핑몰/가격비교/리뷰에서 확인
낮음	블로그/커뮤니티/간접 언급
추정	데이터가 부족해 가능성으로 해석

표기 예시

확실도: 높음
근거: A몰 공식 상품페이지에서 가격 직접 확인
확실도: 중간
근거: 네이버 쇼핑 검색 결과에서 확인했으나 쿠폰가 적용 여부 불확실
확실도: 낮음
근거: 블로그 리뷰 2건에서 반복 언급
확실도: 추정
근거: 검색량 상승과 리뷰 문구를 바탕으로 한 해석

21. Perplexity 인용 환각 방지 규칙

Perplexity나 검색형 모델이 제공한 인용은 그대로 믿지 않는다.

반드시 주의할 점

- URL이 실제여도 본문 내용이 다를 수 있다.
- 숫자, 가격, 날짜는 원문 재확인이 필요하다.
- 여러 출처의 내용을 섞어서 말할 수 있다.
- 인용된 문장이 실제 페이지에 없을 수 있다.
- 검색 결과 스니펫과 실제 페이지 내용이 다를 수 있다.

방어 규칙

- 핵심 수치에는 “원문 확인 필요”를 붙인다.
- 가격/할인/날짜는 확인 시각을 반드시 기록한다.
- 불확실한 내용은 사실처럼 쓰지 않고 “추정”으로 표기한다.
- 출처 URL은 사장 검증용으로 저장한다.
- 최소 2개 이상 출처가 있으면 교차 확인 표시를 한다.

⸻

22. 크롤링 및 수집 제약

모니터링 직원은 무단 크롤링이나 로그인 기반 수집을 하지 않는다.

허용

- 공개 웹페이지 확인
- 사장이 붙여넣은 경쟁사 URL 분석
- 공개 쇼핑 검색 결과 참고
- 공개 리뷰 요약
- 공개 트렌드 도구 참고
- 사장이 제공한 텍스트/캡처 분석

금지

- 로그인 필요한 내부 데이터 무단 접근
- 비공개 회원 정보 수집
- 대량 자동 크롤링
- 경쟁사 시스템 우회
- 유료/제한 데이터 무단 복제

한계 표기 예시

본 리포트는 공개 웹과 사장 제공 자료 기준입니다.
스마트스토어 쿠폰가, 쿠팡 노출 순위, 개인화 검색 결과, 로그인 후 혜택은 누락될 수 있습니다.
가격은 확인 시각 기준이며 이후 변동될 수 있습니다.

⸻

23. 데이터가 부족할 때의 처리

빈손으로 끝내면 안 된다.
데이터가 부족하면 무엇을 더 수집해야 하는지 알려준다.

잘못된 답변

자료가 부족해서 분석할 수 없습니다.

좋은 답변

현재 경쟁사 URL이 없어 실제 가격 변화는 확인할 수 없습니다.
대신 다음 자료가 있으면 다음 리포트에서 변화 감지가 가능합니다.
필요 자료:
1. 주요 경쟁사 3~5개 URL
2. 우리 상품 가격/구성
3. 지난주 또는 이전 확인 가격
4. 주요 키워드 5개
5. 비교하고 싶은 채널

27. BASE_SOP.monitor 압축본

실제 코드에 넣을 압축 SOP는 아래처럼 정리한다.

You are a monitoring analyst. Your outputKind is monitor_digest.
Goal:
Detect meaningful changes in competitors, market, and category trends. Do not merely list information. Always answer: what changed, why it matters, and what we should do.
Principles:
- Monitoring is change detection, not information dumping.
- Put action-required items at the top.
- Compare competitors against our product, price, composition, USP, and positioning.
- Every factual claim should include source URL, checkedAt, sourceType, and confidence.
- Do not present guesses as facts. Mark uncertain points as assumptions or estimates.
- If data is insufficient, explain what data should be collected next.
- Do not perform unauthorized crawling, login-based access, or private data collection. Use public web, price comparison, public reviews, search trends, or owner-provided URLs/text only.
Track:
Competitors:
- price
- discount/coupon/shipping
- composition/options
- new products
- promotions
- review strengths
- review weaknesses
- search/ad exposure if available
- detail page message
- ad copy
Market/category:
- search trend
- rising keywords
- seasonality
- new entrants
- customer language
- category price range
- material/style trends
Priority:
- Red: immediate action. Examples: competitor price drop, strong new entrant, review complaints spike, major promotion, ranking surge.
- Orange: review. Examples: new product, promotion start, message shift, meaningful keyword rise.
- Green: reference. Examples: gradual trend, seasonal signal, general category movement.
- Gray: no meaningful change.
Required output:
1. Summary
2. Action required section
3. Competitor comparison table
4. Change log
5. Trend section
6. Response strategy
7. Source list
8. Limitations
9. Handoff
For each alert:
- level
- title
- change
- impact
- recommendation
- source URL
- checkedAt
- sourceType
- confidence
- status
Comparison:
- Compare our product vs competitors.
- Include price, composition, benefits, review strengths, review weaknesses, difference, and recommended response.
- If sets/options differ, calculate unit price or explain why direct comparison is limited.
- Separate regular price, sale price, coupon price, and shipping when available.
Trend:
- Include keyword, delta, meaning, recommended use, and citation.
- Do not overstate search trend data. Treat it as directional unless exact data is verified.
Strategy:
Suggest response through:
- pricing
- composition
- messaging
- ads
- SNS
- detail page
- sourcing/product planning
- analyst follow-up
- owner approval
Citation discipline:
- Include URL, checkedAt, sourceType, checked item, confidence, and notes.
- Be careful with citation hallucination from search models.
- Core numbers such as price, discount, and dates require original source verification.
- If not verified, mark as estimate or needs owner verification.
Limitations:
Always mention when relevant:
- prices can change after checkedAt
- coupon/login/personalized benefits may be missing
- marketplace rankings can vary by user/location/device
- public reviews may not represent all customers
- some platforms may block or limit public visibility
Handoff:
- sourcing: competitor product/configuration opportunities
- ads: response copy angle
- SNS: trend/content topics
- detail page: USP and landing consistency improvements
- analyst: metric comparison and performance tracking
- owner: price, offer, and paid response approval

## 1. SOP 내용·모델 점검

### 1-1. 철학 — “어제와 뭐가 달라졌나”

- “정보 나열이 아니라 변화 감지”, “무엇이 달라졌고 그래서 뭘 해야 하나”는 요즘 경쟁 분석/CI 툴들이 강조하는 **델타·인사이트 중심 리포팅** 철학과 일치합니다. [pagetest](https://pagetest.ai/blog/best-practices-ai-competitor-analysis-2026)
- “사장은 매일 경쟁사를 볼 수 없고, 이 직원은 달라진 것만 위로 올린다”는 표현은 실제 Perplexity+워크플로 엔진(n8n 등)으로 **주간 경쟁 브리프를 자동 발행하는 패턴**과 목적이 같습니다. [pertamapartners](https://www.pertamapartners.com/workflow-guides/perplexity-ai-competitive-intelligence-workflow)

→ 철학은 그대로 BASE_SOP 상단에 짧게 들어가면 충분합니다.

***

### 1-2. 추적 항목 — 경쟁사·시장 둘 다 포함한 점

- 경쟁사에 대해 가격·구성/옵션·리뷰 키워드(강/약점)·신상품·프로모션·노출 순위를 추적하는 구조는, 상용 경쟁 분석/대시보드 툴들이 제공하는 핵심 메트릭과 거의 같고, [figma](https://www.figma.com/resource-library/ai-competitor-analysis-tools/)
- 시장/카테고리에 대해 검색량 추이·급상승 키워드·시즌성·신규 진입자까지 보는 것도, **키워드·콘텐츠 갭·신규 플레이어**를 찾는 최신 경쟁 분석 패턴과 잘 맞습니다. [trysight](https://www.trysight.ai/blog/competitive-intelligence-best-practices)

→ “경쟁사(수평) + 카테고리/검색(수직)” 두 축을 다 보는 게 매우 좋고, BASE_SOP에는 **“반드시 둘 다 포함하라”** 정도만 강조해 두면 됩니다.

***

### 1-3. 변화 감지 레벨링(🔴🟠🟢)과 우선순위

- 변화가 없는 “상태 설명”보다, 중요한 변화에만 알림을 띄우는 것은 최신 브랜드/AI 모니터링 대시보드에서 권장하는 방식입니다. [seenos](https://seenos.ai/ai-brand-monitoring/ai-brand-monitoring-dashboard)
- “대응 필요를 상단 고정” + 🔴(즉시 대응: 가격 인하·강자 진입·리뷰 급변) / 🟠(검토: 가격 인상·신상품·프로모션) / 🟢(참고: 완만한 동향) 구조는, 실제 CI 대시보드의 **Alerts/KPI 카드 → Trends → 상세** 계층과도 잘 어울립니다. [portermetrics](https://portermetrics.com/en/dashboard-templates/competitors-analysis/)

→ 이 3단계 레벨링은 그대로 가져가되, BASE_SOP에 “**🔴 카드가 하나라도 있으면 리포트 상단에 '⚠ 대응 필요' 섹션을 항상 생성한다**” 정도를 못 박아 두면 됩니다.

***

### 1-4. 비교표 + 대응 전략

- 경쟁사 vs 우리 가격·구성·리뷰 강/약점·차별점 비교는, Crayon/Similarweb 등 상용 경쟁 분석툴이 제공하는 기본 비교 뷰와 같은 구조입니다. [redrattlercreative](https://redrattlercreative.com/best-ai-for-market-research-and-analysis/)
- “세트 구성일 때 1인분 단가로 실질 비교”처럼 **실제 구매단위 기준으로 환산해서 비교하는 것**은, 많은 마케팅/광고팀들이 수기로 하는 일이기도 해서 매우 좋은 디테일입니다. [pagetest](https://pagetest.ai/blog/best-practices-ai-competitor-analysis-2026)
- “가격으로? 구성으로? 메시지로? 우리 USP를 살릴 각도 제시”는, Perplexity 기반 CI 워크플로에서도 권장하는 “데이터 기반 SWOT + 실행 전략” 단계와 유사합니다. [trysight](https://www.trysight.ai/blog/competitive-intelligence-best-practices)

→ “비교표 + 전략” 구조는 완전히 정석이고, BASE_SOP에는  
“**결론 없이 비교표로만 끝내지 말고, 최소 2–3줄의 대응 전략을 제시하라**” 정도만 추가해두면 됩니다.

***

### 1-5. 인용 규율·모델·제약

- Sonar 계열은 **실시간 웹 검색 + 인용 URL 반환**이 강점이라, 모니터링/경쟁 리포트에서 거의 표준처럼 쓰이고 있습니다. [docs.perplexity](https://docs.perplexity.ai/docs/cookbook/showcase/executive-intelligence)
- 다만, Perplexity의 인용 URL이 실제 페이지여도 문장 레벨에서 환각/과장·요약 오류가 있을 수 있어서, **URL+확인 시각+출처 유형을 남기고 핵심 수치는 사람이 검증**하는 패턴이 많이 권장됩니다. [keyword](https://keyword.com/blog/track-brand-mentions-perplexity-ai/)
- “공개 웹/가격비교/사장이 붙여넣은 URL만 사용, 무단 크롤링·로그인 금지, 스마트스토어/쿠팡 정보 일부 누락 가능성 명시”는 실제 AI 기반 경쟁 분석 툴들도 내세우는 합법/컴플라이언스 가드레일과 같은 방향입니다. [brandbeacon](https://www.brandbeacon.ai/platform/perplexity-search-tracking)

→ Perplexity Sonar Pro로 수집+인용, Claude Sonnet으로 정규화·전략을 만드는 구조가 지금 업계에서 추천되는 “실시간 조사→정리” 패턴과 정확히 일치합니다. [perplexity](https://www.perplexity.ai/hub/blog/perplexity-sonar-dominates-new-search-arena-evolution)

## 2. BASE_SOP.monitor에 넣을 핵심 요약

지금 긴 SOP에서 BASE_SOP에 꼭 넣어야 하는 건 다음 정도입니다.

1. **역할·철학 2–3줄**  
   - “너는 경쟁사·시장 모니터링 담당이다. 정보 나열이 아니라 ‘어제와 무엇이 달라졌는지’와 ‘그래서 우리가 뭘 해야 하는지’를 요약한다. 변화가 없으면 조용히 있고, 변화가 있을 때만 위로 올린다.”  

2. **추적 항목 정의**  
   - 경쟁사: 가격, 구성/옵션, 리뷰 키워드(강/약점), 신상품, 프로모션, 노출 순위.  
   - 시장/카테고리: 검색량 추이, 급상승 키워드, 시즌성, 신규 진입자. [redrattlercreative](https://redrattlercreative.com/best-ai-for-market-research-and-analysis/)

3. **변화 감지·우선순위**  
   - 🔴 즉시 대응: 가격 인하, 신규 강자 진입, 리뷰 불만 급증 등.  
   - 🟠 검토: 가격 인상, 신상품 출시, 프로모션 시작 등.  
   - 🟢 참고: 검색량 완만 변화나 일반 동향.  
   - 🔴/🟠가 있으면 항상 상단에 “⚠ 대응 필요” 섹션 생성.  

4. **출력 구조(monitor_digest)**  
   - 상단 Alerts(🔴/🟠 리스트 + 한 줄 코멘트).  
   - 경쟁사 비교표(가격/구성/리뷰 강·약/1인분 단가).  
   - 트렌드 요약(검색량 변화·급등 키워드).  
   - 대응 전략(가격/구성/메시지 중 무엇으로 대응할지 + 우리 USP 각도).  
   - 핸드오프(소싱/광고/SNS/상세에 넘길 포인트).  

5. **인용 규율·제약**  
   - 주장에는 항상 URL+확인 시각+출처 유형(공식몰/쇼핑몰/뉴스/커뮤니티 등)을 붙이고, 불확실한 건 “추정”이라고 표기.  
   - 무단 크롤링·로그인 금지, 공개 웹/가격비교/사장 제공 URL만 사용, 스마트스토어/쿠팡 일부 정보 누락 가능성을 명시. [espressio](https://espressio.ai/blog/n8n-perplexity-competitive-intel-agent/)

6. **모델 파이프라인**  
   - Sonar Pro/Reasoning으로 수집·인용, Claude Sonnet으로 구조화·요약·전략 작성. [docs.perplexity](https://docs.perplexity.ai/docs/cookbook/showcase/executive-intelligence)

이 정도만 BASE_SOP에 넣고, 지금 긴 문서는 내부 레퍼런스로 유지하면 좋습니다.

## 3. 모니터링 직원 페이지 레이아웃 제안 (`monitor_digest`)

이제 “모니터링” 직원 페이지를 **실제 CI·브랜드 모니터링 대시보드**의 정보 구조에 맞춰 설계해 볼게요. [livesov](https://livesov.com/perplexity-brand-tracking)

핵심은:  
1) **알림/변화(Alerts)** → 2) **비교·트렌드(Details)** → 3) **대응 전략/핸드오프(Action)** 순으로 위에서 아래로 내려가는 구조입니다. [seenos](https://seenos.ai/ai-brand-monitoring/ai-brand-monitoring-dashboard)

### 3-1. 상단 헤더

**왼쪽**

- `📡 모니터링 · 경쟁사·트렌드 주간 리포트`  
- 한 줄 설명: `어제와 달라진 것만 골라서, 우리가 해야 할 일까지 제안하는 직원`  
- 현재 워크스페이스/브랜드 이름 (시목 등)  

**오른쪽**

- 모델 뱃지: `Perplexity Sonar Pro (수집·인용) + Claude Sonnet (정규화·전략)` [perplexity](https://www.perplexity.ai/hub/blog/perplexity-sonar-dominates-new-search-arena-evolution)
- 상태 토글: `활성 / 일시정지`  
- 버튼: `[📝 프롬프트] [⚙️ 추적 키워드·경쟁사 설정] [🗑 해고]`  

그 아래 **요약 배너**:

- `이번 주 모니터링: 경쟁사 4곳 · 🔴 즉시 대응 1건 · 🟠 검토 3건`  
- 후에 분석가와 연동 시: `지난 4주 가격/리뷰 변화 요약` 정도도 추가 가능. [pagetest](https://pagetest.ai/blog/best-practices-ai-competitor-analysis-2026)

***

### 3-2. 상단 메인 – Alerts 패널 (“⚠ 대응 필요”)

페이지 최상단 메인 영역에는 **알림 패널**을 둡니다. (대시보드의 Alerts 위젯과 동일한 위치) [portermetrics](https://portermetrics.com/en/dashboard-templates/competitors-analysis/)

**구성**

- 큰 타이틀: `⚠ 대응 필요`  
- 안에 카드 리스트:

예시 카드:

- `🔴 A몰 티크도마 42,000→37,000 인하 (6/14 확인 · 쇼핑몰)`  
  - 하위 텍스트: `우리 39,000과 가격 차이 2,000원 → 가격 추격은 마진 위험, 메시지/구성으로 대응 권장.`  
- `🟠 D몰 신규 진입, "친환경" 메시지 전면 (6/13 · 공식몰)`  

각 카드 우측에는:

- `[자세히 보기]` (아래 비교표 섹션으로 스크롤/포커스)  
- `[광고에 넘기기]`, `[SNS에 넘기기]`, `[상세페이지에 인사이트로]` 같은 핸드오프 버튼.  

→ 사장은 페이지에 들어오자마자 “지금 반응해야 할 변화”를 가장 먼저 보게 됩니다. [seenos](https://seenos.ai/ai-brand-monitoring/ai-brand-monitoring-dashboard)

### 3-3. 중간 – 좌측 “관심 키워드/경쟁사” 패널 + 우측 monitor_digest 상세

화면을 가로로 나누어:

- **좌측**: 설정/관리 패널 (관심 키워드, 추적 경쟁사 리스트)  
- **우측**: 특정 실행(날짜 기준)의 `monitor_digest` 상세 뷰  

#### (A) 좌측 — 추적 설정/필터

- **관심 키워드 섹션**  
  - 예: `["티크 도마", "통원목 도마", "원목 도마 관리", "신혼 주방"]`  
  - 각 키워드 옆에 최근 검색량 방향(↑, →, ↓) 아이콘. [keyword](https://keyword.com/blog/track-brand-mentions-perplexity-ai/)

- **추적 경쟁사 섹션**  
  - 리스트: `A몰 스마트스토어`, `B몰 쿠팡`, `C몰 자사몰`, `D몰 네이버 스마트스토어`  
  - 각 항목 옆에: `채널(스마트스토어/쿠팡/자사몰) · 최근 스캔 날짜`  

- **기간/리포트 선택**  
  - 드롭다운: `이번 주 / 지난 주 / 날짜별 리포트`  
  - 선택에 따라 우측 `monitor_digest` 내용이 바뀜.  

#### (B) 우측 — monitor_digest 상세 뷰

하나의 리포트를 다음 순서로 보여줍니다.

1. **헤더**

   - `YYYY-MM-DD 주간 모니터링 리포트`  
   - `스캔 기준 시각: 08:00 · Sonar Pro` [pertamapartners](https://www.pertamapartners.com/workflow-guides/perplexity-ai-competitive-intelligence-workflow)

2. **Alerts 요약 (상단 패널과 동일, 여기서는 전체 리스트)**  

3. **경쟁사 비교표 섹션**

   - 테이블:

     |        | 가격     | 구성          | 리뷰 강점         | 리뷰 약점           | 1인분 단가 |
     |--------|----------|--------------|-------------------|--------------------|-----------|
     | A몰    | 37,000   | 단품         | 디자인 예쁨       | 관리 어렵다        | 37,000    |
     | 우리   | 39,000   | +관리 카드   | 통원목·국내 마감 | (신규)             | 39,000    |

   - 각 행 클릭 시, 오른쪽에 해당 경쟁사 상세(최근 리뷰 키워드, 신상품, 프로모션 요약) 카드 노출.  
   - “1인분 단가”는 복수 구성일 때 의미 있게 표시. [trysight](https://www.trysight.ai/blog/competitive-intelligence-best-practices)

4. **트렌드 섹션**

   - `트렌드` 제목 아래, 키워드 카드:

     - `"통원목 도마" · 검색량 +18% (전주 대비) · 상위 인용 3개 링크`  
     - `"원목 도마 관리" · 검색량 +12%` 등. [redrattlercreative](https://redrattlercreative.com/best-ai-for-market-research-and-analysis/)

   - 각 키워드 카드에:  
     - `어떤 유형 콘텐츠에서 많이 보이는지`(블로그/커뮤니티/뉴스 등) 간단 표기. [pagetest](https://pagetest.ai/blog/best-practices-ai-competitor-analysis-2026)

5. **대응 전략 섹션**

   - `대응 전략` 제목 아래:

     - `A몰 약점인 "관리 어려움"을 활용해, 우리는 '관리 쉬운 통원목'(천연오일+관리 카드) 메세지를 전면으로.`  
     - `가격 추격 대신, 관리법 콘텐츠 강화+상세페이지 섹션 보강.`  
     - `신규 D몰의 "친환경" 포지셔닝에 대해, 우리는 "오래 쓰는 지속가능성" 각도로 차별화.`  

   - 하단에 핸드오프 버튼:

     - `[광고: 이 소구점으로 카피 세트 생성]`  
     - `[SNS: 트렌드 콘텐츠 주제로 보내기]`  
     - `[상세페이지: 관리법/USP 섹션 보강 인사이트로 넘기기]`  

6. **출처 섹션**

   - 하단에 Sonar 응답의 상위 인용 URL 3–5개만 노출:

     - `https://smartstore.naver.com/... (A몰 가격, 6/14 08:00 기준 · 쇼핑몰)`  
     - `https://blog.naver.com/... (원목 도마 관리 팁, 블로그)`  
     - `https://search.naver.com/... (검색량/키워드 트렌드, 검색엔진)` [espressio](https://espressio.ai/blog/n8n-perplexity-competitive-intel-agent/)

   - “※ 가격·구성·리뷰 내용은 옵션/쿠폰/기간에 따라 달라질 수 있으므로, 중요한 의사결정 전에는 반드시 원문 확인을 권장합니다.” 같은 문구로 한계를 명시. [livesov](https://livesov.com/perplexity-brand-tracking)

### 3-4. “직접 시키기” 패널 (사장 수동 분석)

상단에 다른 직원들처럼 **“직접 시키기” 패널**을 둡니다.

- 타이틀: `직접 시키기 (특정 경쟁사·아이템 분석)`  
- 입력 필드:  
  - 경쟁사 URL (스마트스토어/쿠팡/자사몰 등)  
  - (또는) 텍스트 붙여넣기 (상세페이지/상품 설명)  
  - 키워드(선택)  
  - 분석 범위 선택: `가격/구성/리뷰 / 신상품 / 프로모션`  

- [분석 실행] 버튼  

실행 시:

1. Sonar Pro가 해당 URL/키워드 기반으로 경쟁사 정보를 수집하고 인용과 함께 반환. [docs.perplexity](https://docs.perplexity.ai/docs/cookbook/showcase/executive-intelligence)
2. Claude가 이를 `monitor_digest`의 “경쟁사 비교·트렌드·전략” 구조에 맞게 압축.  
3. 우측 `monitor_digest` 뷰에 즉시 결과 표시 + `daily_reports`에 manual로 저장.  

→ 사장이 특정 페이지/이벤트를 바로 던져 “이번 경쟁사 이 페이지, 우리랑 뭐가 다른지 빨리 정리해줘” 같은 워크를 할 수 있습니다.

### 3-5. 히스토리 탭·알림 설정

상단 탭:

- `[📊 오늘 리포트] [📚 히스토리] [🔔 알림·임계치 설정]`  

#### (A) 히스토리 탭

- 테이블:

  - `날짜 · 주요 🔴/🟠 신호 개수 · 핵심 요약(한 줄) · 관련 경쟁사`  
- 행 클릭 시 해당 날짜의 `monitor_digest` 로딩.  
- 기간 필터(1주/1달/3달) + 키워드/경쟁사 검색.  

→ 장기적으로 “가격 인하가 얼마나 자주 있었는지”, “새 경쟁사 진입 패턴” 등을 돌아볼 수 있습니다. [portermetrics](https://portermetrics.com/en/dashboard-templates/competitors-analysis/)

#### (B) 알림·임계치 설정 탭

- **경고 임계치 설정**

  - 예:  
    - `가격 인하 폭 >= 10%` → 🔴  
    - `리뷰 불만 키워드 증가율 >= 30%` → 🔴  
    - `검색량 전주 대비 >= 20% 상승` → 🟠  

- **알림 채널 설정**

  - `앱 내 알림 / 이메일 / 슬랙` (나중에 연동 시)  

→ 이렇게 하면 모니터링 직원이 **정해진 기준 이상일 때만 사장에게 알림**을 보내도록 제어할 수 있어, 노이즈를 줄일 수 있습니다. [keyword](https://keyword.com/blog/track-brand-mentions-perplexity-ai/)

## 4. 정리

- 모니터링 SOP는 **변화 중심 철학, 추적 항목, 우선순위 레벨(🔴/🟠/🟢), 경쟁사 비교·대응 전략, 인용 규율, Sonar Pro→Claude 파이프라인, 크롤링 제약**까지, 현재 AI 기반 경쟁 분석/브랜드 모니터링 베스트프랙티스와 매우 잘 맞습니다. [perplexity](https://www.perplexity.ai/hub/blog/perplexity-sonar-dominates-new-search-arena-evolution)
- BASE_SOP.monitor에는 이 중 “역할·철학 / 추적 항목 / 우선순위 기준 / monitor_digest 출력 포맷 / 인용 규율·제약 / 모델 파이프라인”만 압축해 넣고, 나머지 디테일은 SOP 문서에 두면 재사용성이 좋습니다. [espressio](https://espressio.ai/blog/n8n-perplexity-competitive-intel-agent/)
- 모니터링 직원 페이지는  
  - 상단 헤더 + 이번 주 요약,  
  - 최상단 Alerts 패널(⚠ 대응 필요),  
  - 좌측 추적 키워드·경쟁사 + 우측 monitor_digest 상세(Alerts→비교표→트렌드→전략→출처),  
  - 상단 “직접 시키기” 패널,  
  - 히스토리·알림 설정 탭  
  구조로 잡으면, 실제로 **주간 경쟁 인텔 브리프를 자동으로 올려주는 “CI 담당 직원”처럼** 자연스럽게 사용할 수 있을 것입니다. [livesov](https://livesov.com/perplexity-brand-tracking)
