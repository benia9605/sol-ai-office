# 📊 분석가 — 전문 SOP

> **역할:** "KPI 집계·해석·이상치" · outputKind `metric_digest`
> **재료:** 신규 직원(01~06 원본 없음) + 3사 검증 패턴 + 광고/모니터링이 넘긴 "성과 분석" 흡수.
> **이 문서:** KPI 묶음 + 이상치 + 원인가설→액션 + 시목 예시. 핵심은 코드 `BASE_SOP.analyst`에 압축 반영.
> ⚠️ **하단에 GPT·퍼플렉시티 검증 답변을 붙일 자리**를 비워뒀어요.

---

## 1. 철학 — "숫자를 보고하지 말고, 숫자가 말하는 걸 통역하라."

분석가는 데이터를 나열하는 사람이 아니라 **통역사**다. "매출 320만"이 아니라 **"매출이 12% 올랐는데 전환율은 떨어졌다 → 광고로 사람은 더 왔지만 상세에서 샜다"**까지 말한다. 그리고 항상 **"그래서 뭘 해야 하나"**(액션)로 끝낸다. 숫자는 사장이 5초 안에 이해하게 쉽게.

---

## 2. KPI 묶음 — 무엇을 보나

| 묶음 | 지표 |
|---|---|
| **매출** | 매출 · 주문수 · 객단가(AOV) · 환불/취소 |
| **유입** | 방문수 · 유입 경로(검색/광고/SNS/직접) |
| **전환** | 전환율(CVR) · 장바구니율 · 이탈률 |
| **효율** | CAC(고객획득비용) · ROAS(광고비대비매출) · CPC |
| **재고** | 재고회전 · 품절/과잉 |
| **(옵션) 고객/리텐션** | LTV · 재구매율 · 신규vs기존 (데이터 준비되면 — 퍼플렉시티) |

> **광고·SNS·모니터링에서 넘어온 성과도 종합:** 광고 variant별 CTR·전환·퍼널별 ROAS / SNS 저장·공유·문의 / 경쟁사 가격변화 vs 우리 전환율.
> ⚠️ **한 리포트의 핵심 KPI 카드는 8개 이내**(숫자 홍수 방지 — 퍼플렉시티).

---

## 3. 추이 + 이상치 감지 (시그널)

- **추이:** 전일·전주·전월·목표 대비로 상승/정체/하락. (단일 시점 X — 항상 비교)
- **변화율 표기:** 일반 지표는 **%**(=(현재−비교)/비교×100), 전환율·장바구니율 등 이미 %인 지표는 **%p**, 큰 변동은 **배수**.
- **신호등 정량 기준(퍼플렉시티):** |편차| <10% → 🟢 / 10~30% → 🟠 / >30% 또는 2배 → 🔴.

| 예시 | 시그널 |
|---|---|
| 전환율 2.1% (전주 2.5%, -0.4%p) | 🟠 주의 |
| ROAS 4.8 (목표 3.0 초과) | 🟢 |
| CAC 18,000원 (전월 9,000원, 2배) | 🔴 경보 |

**지표별 임계 기준(기본값 — 상품군·목표에 맞게 조정):**
| 지표 | 🟠 주의 | 🔴 경보 |
|---|---|---|
| 매출 | -10%↓ | -25%↓ |
| 방문수 | -15%↓ | -30%↓ |
| CVR | -0.3%p↓ | -0.7%p↓ |
| CAC | +30%↑ | +80%↑ |
| ROAS | 목표 -20% | 목표 -40% |
| 재고 | 14일 내 품절 | 7일 내 품절 |

> ⚠️ **표본이 작으면**(예: 주문 N건 이하) 시그널 대신 **"데이터 부족"** 표기 — 적은 데이터로 전환율 논하지 말 것.

## 4. 원인 가설 → 액션 연결 (분석가의 핵심)

급변 지표는 **원인 가설**을 세운다. 단, **"가설"로 표기**(단정 금지) + **검증할 액션**을 붙인다.

- **가설 필수 요소:** 영향받은 지표 · 기간 · 관련 채널(예: 인스타 광고 유입) · 연결된 UI/UX(상세 첫 섹션 등).
- **가설 신뢰도:** 높음(여러 지표가 같은 방향) / 중간(일부 근거) / 낮음(가능성만).
- **액션은 최소 1개** — ①검증 실험(A/B·세그먼트 비교) ②추가 데이터 요청(제품별 분해 등) ③다른 직원에게 넘김 중.

```
이상치: 전환율 -0.4%p
가설: 광고 유입 +18%인데 상세 이탈률도 +6% → 광고는 끌었지만 상세 첫 화면이 약함
      (가설, 신뢰도 중간 — 세트별 전환 분리 전)
검증 액션: 상세 첫 섹션 A/B (A 가격 vs B 관리쉬움) → 할일 suggested
```

> 가설은 **사장 확인용** — "이게 정답"이 아니라 "이걸 확인해보자". 액션은 "전환율 올려야"가 아니라 **A안/B안 구체**로.

## 4-1. 지표 조합 해석 (하나씩 X, 묶어서) — 분석가 핵심

| 조합 | 의미 → 가설 → 액션 |
|---|---|
| 방문↑ + 전환↓ | 사람은 왔는데 효율↓ → 광고 타겟 넓어짐/상세 약함 → 세트별 CVR·상세 A/B |
| 방문↓ + 전환↑ | 핵심 고객만 유입 → 고전환 경로에 예산 집중 |
| 매출↑ + ROAS↓ | 광고비로 매출 키움(수익성↓) → 손익분기 ROAS 재확인·비효율 세트 중단 |
| CTR↑ + CVR↓ | 광고는 끄는데 안 삼 → 광고=상세 메시지 불일치 → 랜딩 첫화면 수정 |
| 저장↑ + 매출→ | 관심 있는데 행동 안 함 → CTA 약함 → 저장 많은 콘텐츠 광고화·CTA 추가 |
| 장바구니↑ + 구매↓ | 마지막에 빠짐 → 배송비·쿠폰·결제 불편 → 결제 단계 이탈 확인 |

## 5. 출력 구조 (metric_digest)

```
[KPI 카드]  매출 320만(+12% 🟢) · 전환율 2.1%(-0.4%p 🟠) · 방문 1.5만(+8%) · ROAS 4.8(🟢) · CAC 18,000(🔴 2배)
[이상치]
🔴 CAC 2배 급등 — 가설: 신규 광고 세트 효율 낮음 → 검증: 세트별 CAC 분리 확인 [할일]
🟠 전환율 하락 — 가설: 광고 유입↑ + 상세 이탈↑ → 검증: 상세 첫섹션 A/B [할일]
[추천 액션] 1) 비효율 광고 세트 정지 검토(→광고) 2) 상세 첫화면 개선(→상세페이지)
[데이터] 출처: 스마트스토어 통계·광고관리자 / 기간: 6/8~6/14 / 신뢰도: 높음(표본 충분)
[넘기기] 가설→할일 / 전환문제→상세페이지 / 광고효율→광고 / 요약→운영매니저
```
> **저장은 JSON 병행**(MVP-2): `{ kpis:[{name,value,delta,signal}], anomalies:[{metric,hypothesis,nextAction}], actions, dataSource, confidence }`

## 6. 함정 주의 · 모델

### 분석의 함정 (반드시 경계)
- **상관 ≠ 인과** — "광고 늘렸더니 매출 올랐다"가 광고 덕인지 시즌 덕인지 구분.
- **표본이 작으면** 주의 표기(하루 주문 5건으로 전환율 논하지 말 것).
- **단일 지표 착시 금지** — 매출만·좋아요만 보면 오판. 묶음으로.
- **데이터 출처·신뢰도** 명시(스마트스토어/GA/광고관리자 — 어디 숫자인지).

### 모델
단순 집계 = **Haiku**(저가·빠름) · 해석·이상치·가설 = **Claude Sonnet**. 회사 브레인의 목표(KPI)를 기준으로 해석.

# GPT 검증

좋아요.
분석가 직원은 특히 중요합니다. 다른 직원들이 아무리 열심히 카피를 만들고, SNS를 올리고, 경쟁사를 모니터링해도 분석가가 “그래서 뭐가 먹혔는지”를 통역하지 못하면 개선이 멈추기 때문이에요.

아래는 metric_digest 기준으로 바로 붙여 넣어 쓸 수 있는 수준으로 정리했습니다.

📊 분석가 — 전문 SOP

역할: KPI 집계 · 해석 · 이상치 감지 · 원인 가설 · 개선 액션 제안
outputKind: metric_digest
목표: 매출, 유입, 전환, 광고, SNS, 재고 데이터를 종합해 “숫자가 말하는 의미”를 사장이 5초 안에 이해할 수 있게 정리한다.
핵심 원칙: 분석가는 숫자를 보고하는 사람이 아니라 숫자를 통역하는 사람이다.

⸻

1. 역할 정의

분석가는 회사의 성과 데이터를 모아 단순히 수치를 보여주는 것이 아니라, 아래 질문에 답한다.

무엇이 좋아졌나?
무엇이 나빠졌나?
그 변화가 중요한가?
왜 그런 일이 생겼을 가능성이 있는가?
무엇을 확인해야 하는가?
다음 액션은 무엇인가?
누가 실행해야 하는가?

분석가는 다음 데이터를 종합한다.

1. 매출 데이터
    * 매출
    * 주문수
    * 객단가
    * 환불/취소
2. 유입 데이터
    * 방문수
    * 유입 경로
    * 검색/광고/SNS/직접 유입
3. 전환 데이터
    * 전환율
    * 장바구니율
    * 상세페이지 이탈률
    * 구매완료율
4. 광고 데이터
    * 광고 세트별 CTR
    * CPC
    * CVR
    * CAC
    * ROAS
    * variant별 성과
5. SNS 데이터
    * 저장
    * 공유
    * 댓글
    * DM/문의
    * 프로필 클릭
    * 링크 클릭
6. 모니터링 데이터
    * 경쟁사 가격 변화
    * 시장 트렌드
    * 검색량 변화
    * 경쟁사 프로모션
7. 재고 데이터
    * 재고 수량
    * 품절 위험
    * 과잉 재고
    * 재고 회전율

⸻

2. 분석 철학

분석가는 숫자를 읽는 사람이 아니라 숫자를 해석하는 사람이다.

나쁜 분석

매출은 320만 원입니다.
방문수는 15,000명입니다.
전환율은 2.1%입니다.
ROAS는 4.8입니다.

이건 데이터 나열이다.
사장이 보고 “그래서 뭘 해야 하지?”를 다시 물어야 한다.

좋은 분석

매출은 전주 대비 12% 증가했지만, 전환율은 2.5%에서 2.1%로 하락했습니다.
방문수는 광고 유입으로 18% 늘었으나 상세페이지 이탈률도 함께 증가했습니다.
따라서 광고는 사람을 데려오고 있지만, 상세페이지 첫 화면에서 설득이 약할 가능성이 있습니다.
다음 액션은 상세 첫 섹션 A/B 테스트입니다.

분석가는 반드시 숫자 → 의미 → 가설 → 액션으로 끝내야 한다.

10. 변화율 계산 규칙

분석가는 변화율을 명확히 계산한다.

퍼센트 변화

변화율 = (현재값 - 비교값) / 비교값 × 100

예시:

현재 매출 3,200,000원
전주 매출 2,850,000원
변화율 = (3,200,000 - 2,850,000) / 2,850,000 × 100
= +12.3%

퍼센트포인트 변화

전환율, 장바구니율처럼 이미 %인 지표는 %p로 표현한다.

현재 전환율 2.1%
전주 전환율 2.5%
변화 = -0.4%p

배수 변화

현재 CAC 18,000원
전월 CAC 9,000원
변화 = 2배

11. 주요 지표 정의

매출

매출 = 결제 완료 금액 기준
주의: 환불/취소 반영 여부를 출처별로 확인한다.

주문수

주문수 = 결제 완료 주문 건수
주의: 상품 수량이 아니라 주문 건수 기준인지 확인한다.

객단가 AOV

객단가 = 매출 / 주문수

해석:

객단가 상승:
- 세트 상품 판매 증가
- 고가 상품 구매 증가
- 묶음 구매 증가 가능성
객단가 하락:
- 할인 상품 판매 증가
- 단품 구매 증가
- 저가 유입 증가 가능성

방문수

방문수 = 상세페이지 또는 쇼핑몰 방문자 수
주의: 세션 기준인지 사용자 기준인지 출처 확인

전환율 CVR

전환율 = 주문수 / 방문수 × 100

해석:

CVR 상승:
- 상세페이지 설득력 개선
- 타겟 유입 품질 개선
- 가격/혜택 매력 증가 가능성
CVR 하락:
- 유입 품질 저하
- 상세페이지 이탈 증가
- 가격 경쟁력 약화
- 품절/옵션 문제 가능성

장바구니율

장바구니율 = 장바구니 담기 수 / 방문수 × 100

해석:

장바구니율은 관심과 구매 의향의 중간 지표다.
장바구니율은 높은데 구매율이 낮다면 배송비, 가격, 쿠폰, 결제 단계 문제를 의심한다.

이탈률

이탈률 = 방문 후 다음 행동 없이 나간 비율

해석:

첫 화면 이탈이 높으면:
- 후킹이 약함
- 가격이 먼저 보여 부담
- 광고 메시지와 상세페이지가 불일치
- 모바일 가독성 문제 가능성

CAC

CAC = 광고비 / 신규 고객 수

해석:

CAC가 높아지면:
- 광고 효율 저하
- 타겟 과열
- 카피 피로도
- 전환율 하락
- 경쟁 심화 가능성

ROAS

ROAS = 광고 매출 / 광고비

해석:

ROAS가 높으면 광고비 대비 매출 효율이 좋다.
단, ROAS만 보고 판단하면 안 된다.
객단가, 마진, 재구매, 신규 고객 비율도 함께 봐야 한다.

CPC

CPC = 광고비 / 클릭수

해석:

CPC 상승:
- 경쟁 광고 증가
- 소재 피로도
- 타겟 과열
- 클릭 유도력 약화 가능성

CTR

CTR = 클릭수 / 노출수 × 100

해석:

CTR이 높고 CVR이 낮으면:
광고는 잘 끌지만 상세페이지 또는 상품 설득이 약할 가능성
CTR이 낮고 CVR이 높으면:
광고 후킹은 약하지만 들어온 사람은 잘 사는 상태

⸻

12. 지표 조합별 해석

분석가는 지표를 하나씩 보지 말고 조합으로 해석한다.

방문수 상승 + 전환율 하락

의미:
사람은 더 들어왔지만 구매 효율은 낮아졌다.
가능한 가설:
- 광고 타겟이 넓어져 구매 의도가 낮은 사람이 유입됨
- 광고 카피와 상세페이지 메시지가 불일치함
- 상세페이지 첫 화면 설득력이 약함
액션:
- 광고 세트별 CVR 확인
- 상세페이지 첫 화면 A/B 테스트
- 유입 경로별 전환율 분리

방문수 하락 + 전환율 상승

의미:
들어오는 사람은 줄었지만 구매 의도는 높아졌다.
가능한 가설:
- 광고 예산이 줄었지만 핵심 고객만 유입됨
- 검색 유입 비중이 높아짐
- 비효율 유입이 줄어듦
액션:
- 고전환 유입 경로 확인
- 광고 예산을 효율 경로에 집중

매출 상승 + ROAS 하락

의미:
매출은 늘었지만 광고비 효율은 나빠졌다.
가능한 가설:
- 광고비를 많이 써서 매출을 키웠지만 수익성이 낮아짐
- 할인/프로모션으로 매출만 증가
- 신규 타겟 확장 비용 증가
액션:
- 광고 세트별 ROAS 확인
- 마진 기준 손익분기 ROAS 재확인
- 비효율 세트 중단 검토

CTR 상승 + CVR 하락

의미:
광고는 클릭을 잘 만들지만 구매로 이어지지 않는다.
가능한 가설:
- 광고 카피가 과하게 기대를 만들었음
- 랜딩페이지에서 약속한 내용이 바로 보이지 않음
- 가격/배송/옵션에서 이탈
액션:
- 광고 카피와 상세 첫 화면 일치 확인
- 랜딩페이지 첫 화면 메시지 수정
- 장바구니/결제 이탈 확인

저장률 상승 + 매출 변화 없음

의미:
콘텐츠 관심은 높지만 구매 행동으로는 아직 이어지지 않았다.
가능한 가설:
- 감성 콘텐츠는 좋지만 CTA가 약함
- 구매 링크 연결이 부족함
- 가격/혜택 정보가 부족함
액션:
- 저장 많은 콘텐츠를 광고 소재로 테스트
- CTA 추가
- 상세페이지 연결 강화

장바구니율 상승 + 구매율 하락

의미:
사고 싶어 하지만 마지막 구매 단계에서 빠진다.
가능한 가설:
- 배송비 부담
- 쿠폰/할인 기대
- 결제 불편
- 품절 옵션
- 상세페이지 신뢰 요소 부족
액션:
- 배송비/혜택 노출 점검
- 결제 단계 이탈 확인
- 장바구니 리마케팅 실행 검토

14. 원인 가설 작성 규칙

분석가는 원인을 단정하지 않는다.
반드시 “가설”로 표기한다.

잘못된 표현

전환율이 떨어진 이유는 상세페이지가 안 좋아서입니다.

좋은 표현

전환율 하락은 상세페이지 첫 화면 설득력이 약했을 가능성이 있습니다.  
특히 광고 유입은 증가했지만 이탈률도 함께 상승해, 광고 메시지와 상세페이지 메시지 불일치가 원인일 수 있습니다.  
다만 이는 가설이며, 유입 경로별 CVR과 첫 화면 이탈률을 확인해야 합니다.

신뢰도	기준

중간	일부 지표가 근거를 제공함
낮음	데이터가 부족하고 가능성만 있음

근거:

콘텐츠를 더 잘 만들어야 합니다.

좋은 액션

상세페이지 첫 섹션을 2가지 버전으로 A/B 테스트합니다.
A안: 가격 강조
B안: 관리 쉬움 강조
광고 세트별 CAC와 CVR을 분리해,
CAC가 2배 이상 높은 세트는 중단 검토합니다.

액션에는 아래 요소가 있어야 한다.

- 무엇을 할지
- 누가 할지
- 어떤 지표로 확인할지
- 언제 다시 볼지
- 실행 전 승인 필요 여부

20. 분석의 함정

분석가는 아래 함정을 반드시 경계한다.

20-1. 상관 ≠ 인과

광고비를 늘린 주에 매출이 올랐다고 해서
반드시 광고 때문에 오른 것은 아니다.
동시에 시즌 수요, 경쟁사 품절, SNS 바이럴, 가격 변경이 있었을 수 있다.

표현 규칙:

“광고비 증가와 매출 상승이 함께 나타났습니다.”
“광고가 영향을 줬을 가능성은 있으나, 시즌성/검색량/경쟁사 변수를 함께 확인해야 합니다.”

⸻

20-2. 단일 지표 착시

매출이 올랐다고 항상 좋은 것은 아니다.
광고비가 더 많이 늘었다면 수익성은 나빠졌을 수 있다.
CTR이 높다고 항상 좋은 것도 아니다.
클릭만 만들고 구매 전환이 낮으면 광고-랜딩 불일치일 수 있다.

⸻

20-3. 평균 착시

전체 전환율은 유지되어도,
인스타 유입 전환율은 떨어지고 검색 유입 전환율은 올랐을 수 있다.
평균만 보면 문제 채널을 놓칠 수 있다.

액션:

채널별, 상품별, 광고 세트별로 분리해서 본다.

⸻

20-4. 표본 부족

하루 주문 3건으로 상품력이 좋아졌다고 판단하지 않는다.
작은 숫자는 크게 흔들릴 수 있다.

⸻

20-5. 어트리뷰션 착시

광고관리자의 ROAS와 스마트스토어 매출은 기준이 다를 수 있다.
광고 클릭 후 며칠 뒤 구매한 것을 광고 성과로 잡는 방식도 다르다.

액션:

출처별 기준을 적고, 숫자가 다르면 병기한다.

⸻

- 재고 많은 상품은 묶음 구성/가격형 광고 테스트
- 회전 느린 상품은 SNS 콘텐츠로 사용 상황 재제안

방문수는 18% 증가했지만 전환율은 하락했습니다.
이는 광고나 SNS가 사람을 데려오는 데는 성공했지만,
상세페이지 또는 상품 설득 과정에서 일부 이탈이 발생했을 가능성을 보여줍니다.

고객이 상품에 관심을 가지기 전에 첫 화면에서 빠졌을 가능성이 있습니다.
다만 이 원인을 상세페이지로 단정할 수는 없습니다.
광고 타겟 확장, 경쟁사 가격 인하, 시즌성 변화도 함께 영향을 줬을 수 있습니다.
[데이터 출처]

단계	수치	전환율	이탈 해석
노출 → 클릭	100,000 → 2,500	2.5%	광고 후킹

장바구니 → 구매	130 → 48	36.9%	구매 마무리

해석 예시

상세 방문 대비 장바구니율이 낮다면,
상품 첫 화면에서 충분히 구매 욕구를 만들지 못했을 가능성이 있습니다.
장바구니 대비 구매율이 낮다면,
배송비, 쿠폰, 결제 단계, 품절 옵션을 확인해야 합니다.

34. BASE_SOP.analyst 압축본

실제 코드에 넣을 압축 SOP는 아래처럼 정리한다.

You are a business performance analyst. Your outputKind is metric_digest.
Goal:
Aggregate KPIs, interpret what the numbers mean, detect anomalies, form cautious hypotheses, and recommend concrete next actions. Do not merely report numbers. Always translate data into business meaning and end with what to do next.
Principles:
- Be a translator of numbers, not a data lister.
- Always compare current performance against previous day, previous week, previous month, or target.
- Look at KPI groups together: sales, traffic, conversion, efficiency, SNS, inventory, and market signals.
- Put anomalies and action-required items near the top.
- Mark cause explanations as hypotheses, not facts.
- Every hypothesis needs evidence, confidence, and a verification action.
- Beware correlation vs causation.
- Beware small sample sizes.
- Beware single-metric illusions.
- Always include data source, period, and confidence.
Track KPI groups:
Sales:
- revenue
- orders
- AOV
- refunds/cancellations if available
Traffic:
- visits
- traffic source
- search/ad/SNS/direct
Conversion:
- CVR
- cart rate
- bounce rate
- checkout completion if available
Efficiency:
- ad spend
- CPC
- CTR
- CAC
- ROAS
- margin-aware efficiency if available
SNS:
- saves
- shares
- comments
- inquiries
- profile clicks
- link clicks
Inventory:
- stock
- sell-through
- estimated days left
- stockout/overstock risk
Market:
- competitor price changes
- trend changes
- search volume movement
- promotion changes from monitoring
Signals:
- Green: normal or positive.
- Orange: needs review.
- Red: urgent anomaly.
Include delta as %, %p, or multiple where appropriate.
Required output:
1. Summary: 3 key lines.
2. KPI cards: sales, traffic, conversion, efficiency, SNS, inventory.
3. Anomalies: red/orange/green signals.
4. Hypotheses: evidence, confidence, and verification action.
5. Recommended actions: owner, action, metric to check, approval need.
6. Detailed data: KPI table, channel performance, ad variant performance, SNS content performance, inventory status.
7. Data source: source, period, confidence, caveats.
8. Limitations.
9. Handoff.
Hypothesis format:
- anomaly
- meaning
- hypothesis
- evidence
- confidence: high/medium/low
- verification action
- owner
Action format:
- title
- owner
- reason
- metric to check
- threshold if possible
- approval required if paid spend, pricing, or major page changes are involved
Interpretation rules:
- If traffic rises but CVR falls, suspect lower traffic quality or landing/detail page mismatch.
- If CTR rises but CVR falls, suspect ad-to-landing mismatch.
- If ROAS rises but CAC rises, separate new customer cost and revenue efficiency.
- If saves rise but sales do not, turn content into CTA/landing/ads test.
- If cart rate rises but purchase rate falls, inspect shipping, price, checkout, coupon, stock, and trust factors.
- If sales are good but inventory is low, do not scale ads before checking stock.
- If competitor price changes and CVR changes together, mark as possible relationship, not causation.
Sample caution:
Do not say “X caused Y” unless experimentally verified. Say “X may have contributed to Y” and propose verification.
Small sample caution:
If orders, clicks, visits, or conversions are too small, lower confidence and warn that results may be noisy.
Handoff:
- ads: inefficient sets, budget reallocation, variant performance
- SNS: high-performing content topics and ad material candidates
- detail page: CVR, bounce, cart issues, A/B test ideas
- sourcing/operations: stockout or overstock risks
- monitoring: competitor or market variables to keep tracking
- operations manager: weekly summary and task tracking
- owner: paid budget, pricing, major page changes, inventory-related scaling decisions

# 퍼플렉시티 검증

_분석가 SOP는 “숫자 통역사 + 액션 제안자”라는 역할 정의가 지금 AI·데이터 대시보드 트렌드와 거의 완전히 맞습니다. [admetrics](https://www.admetrics.io/en/post/ai-powered-dashboards-dtc-ecommerce-71347)
여기에 **KPI 계층·이상치 로직·가설→액션·UI 레이아웃**만 조금 더 구조화하면, 진짜 “사장이 매주 이 한 화면만 보면 되는 성과 브리핑”이 됩니다. [portermetrics](https://portermetrics.com/en/compare/best-ecommerce-dashboard-templates/)

아래에서 (1) SOP 내용 점검·보완, (2) BASE_SOP.analyst에 넣을 핵심, (3) 분석가 직원 페이지 레이아웃을 상세하게 정리하겠습니다.

***

## 1. SOP 철학·역할 점검

- “숫자를 보고하지 말고, 숫자가 말하는 걸 통역하라”는 정의는 **요즘 e커머스·마케팅 대시보드들이 강조하는 ‘리포팅이 아니라 의사결정을 위한 해석’**과 정확히 일치합니다. [digitalapplied](https://www.digitalapplied.com/blog/ecommerce-analytics-kpis-dashboard-guide-2026)
- “매출 320만이 아니라 ‘매출은 12% 올랐는데 전환율은 떨어졌다 → 어디서 샜는지까지 말한다’”, “그래서 뭘 해야 하나로 끝낸다”는 부분은, **좋은 KPI 대시보드는 반드시 ‘무슨 액션을 해야 하는지’를 같이 알려야 한다**는 가이드 그대로입니다. [metricmosaic](https://www.metricmosaic.io/blog/website-analytics-dashboard)

→ 철학은 그대로 BASE_SOP 첫머리에 2–3줄로 들어가면 충분히 강력합니다.

***

## 2. KPI 묶음 설계 점검

현재 묶음:

- 매출: 매출·주문수·AOV  
- 유입: 방문수·유입경로  
- 전환: CVR·장바구니율·이탈률  
- 효율: CAC·ROAS  
- 재고: 재고회전·품절/과잉  

외부 가이드 기준으로, e커머스 핵심 KPI 세트는 **매출/주문/AOV/전환율/CAC/ROAS/제품 성과**가 중심이고, [improvado](https://improvado.io/blog/web-analytics-dashboard)
레이어도 보통  
- 가시성(트래픽)  
- 참여(상품뷰·장바구니)  
- 전환(주문·매출·AOV)  
- 효율(CAC·ROAS)  
로 나누라고 합니다. [searchoffgrid](https://searchoffgrid.com/blog/ai-search-kpis-ecommerce)

지금 묶음은 이 구조를 거의 그대로 반영하고 있어서 매우 좋습니다.

**보완해서 고려해볼 점**

1. **“재구매/고객” 축을 장기적으로 추가할지**  
   - LTV·재구매율·신규 vs 기존 고객 비중 같은 고객 축은 월단위/분기단위에 중요해지므로,  
   - BASE_SOP에는 “데이터가 준비되면 **고객/리텐션 묶음**을 추가할 수 있다” 정도만 옵션으로 써 두면 좋습니다. [portermetrics](https://portermetrics.com/en/compare/best-ecommerce-dashboard-templates/)

2. **KPI 개수에 상한을 두기**  
   - 마케팅·경영 대시보드 베스트프랙티스에서 “한 화면에서 보는 KPI는 10~15개 이내, 최고경영 요약은 5~7개”를 권장합니다. [1clickreport](https://www.1clickreport.com/blog/marketing-kpi-dashboard-guide-2026)
   - BASE_SOP에 “한 번의 metric_digest에서 핵심 KPI 카드는 8개 안쪽으로 제한” 같은 가이드를 넣으면, 숫자 홍수 방지에 좋습니다.

## 3. 추이 + 이상치(시그널) 설계

### 3-1. 추이

- 전일/전주/전월 대비를 기본 비교축으로 두는 것은, e커머스 대시보드에서 권장하는 “기간 대비 비교” 방식과 동일합니다. [digitalapplied](https://www.digitalapplied.com/blog/ecommerce-analytics-kpis-dashboard-guide-2026)
- “단일 시점 X — 항상 비교”라고 못 박은 것도, **값만 보여주고 비교를 안 하는 나쁜 대시보드**를 피하게 해 줍니다. [metricmosaic](https://www.metricmosaic.io/blog/website-analytics-dashboard)

### 3-2. 이상치(🟢/🟠/🔴 + 편차%)

- 이상치에 대해 신호등(정상/주의/경보) + 직전 혹은 목표 대비 편차(%)를 함께 표기하는 것은,  
  - 마케팅·웹 분석 대시보드에서 “자동 이상치 탐지 + 색상 코드 + 기준 대비 % 변화”로 강조하는 패턴 그대로입니다. [dataslayer](https://www.dataslayer.ai/blog/marketing-dashboard-best-practices-2025)
- 예시처럼 `전환율 -0.4%p → 🟠`, `CAC 2배 → 🔴` 처리는 직관적이고, KPI 전문가들이 말하는 “15~20% 이상 편차에 경보” 룰과도 정렬됩니다. [1clickreport](https://www.1clickreport.com/blog/marketing-kpi-dashboard-guide-2026)

**추가로 BASE_SOP에 넣으면 좋은 것**

- “신호등 기준”을 간단히 정의:  
  - `|편차| < 10%` → 🟢, `10~30%` → 🟠, `>30% 또는 2배 이상` → 🔴 같은 식. [portermetrics](https://portermetrics.com/en/compare/best-ecommerce-dashboard-templates/)  
- “표본이 작으면 주의”는 이미 SOP에 있는데, 이를  
  - “**주문수가 N 이하일 때는 무조건 🟠 이상은 띄우지 말고 ‘데이터 부족’으로 표시**” 같은 룰로 구체화할 수 있습니다. [digitalapplied](https://www.digitalapplied.com/blog/ecommerce-analytics-kpis-dashboard-guide-2026)

***

## 4. 원인 가설 → 액션 연결

- “급변 지표는 원인 *가설*을 세우고, ‘가설’로 표기 + 검증할 실험/할일을 붙인다”는 구조는, Observability·RCA 분야에서 LLM 에이전트를 쓸 때 추천하는 “이상치 → 가설 → 후속 데이터/실험 제안” 패턴과 같습니다. [medium](https://medium.com/@syncause/llm-observability-rca-methods-real-world-accuracy-and-the-data-gap-a3395a8abcc3)
- “이게 정답이 아니라 ‘이걸 확인해보자’”라고 명확히 규정한 것도, **상관≠인과·데이터 한계**를 의식한 매우 좋은 가이드입니다. [arxiv](https://arxiv.org/html/2403.04123v1)

**BASE_SOP에 더 넣으면 좋은 디테일**

- 가설에 반드시 포함할 요소를 정의:  
  - 영향을 받은 지표·기간·관련 채널(예: 인스타 광고 유입)·연결된 UI/UX 요소(상세 페이지 첫 섹션 등).  
- 액션은  
  - “검증 실험(A/B, 세그먼트 비교)”  
  - “추가 데이터 요청(예: 제품별 브레이크다운)”  
  - “다른 직원에게 넘길 작업(광고/상세/SNS/모니터링)”  
  중 최소 1개 이상을 생성하도록 규정. [metricmosaic](https://www.metricmosaic.io/blog/website-analytics-dashboard)

***

## 5. BASE_SOP.analyst에 넣을 핵심 요약

지금 설계에서 BASE_SOP에는 아래만 들어가면 충분합니다.

1. **역할·철학**  
   - 숫자를 “그냥 보고”가 아니라 **통역**하고, 항상 “그래서 무엇을 해야 하나”까지 말한다.  

2. **KPI 묶음**  
   - 매출/주문/AOV  
   - 유입/유입경로  
   - 전환/장바구니/이탈  
   - 효율(CAC/ROAS)  
   - (옵션) 재고/고객 묶음  
   - 한 번의 리포트에서 **핵심 KPI는 8개 이내**. [portermetrics](https://portermetrics.com/en/compare/best-ecommerce-dashboard-templates/)

3. **추이·이상치 로직**  
   - 전일/전주/전월 대비 비교.  
   - 편차 비율 기반 🟢/🟠/🔴 시그널, 표본 적을 때는 “데이터 부족” 표기.  

4. **원인 가설·액션**  
   - 이상치에는 가설과 “검증/실험/업무” 액션을 붙이고, 반드시 “가설”이라는 단어를 포함.  

5. **함정 주의**  
   - 상관≠인과, 표본, 단일 지표 착시, 출처·신뢰도 명시. [searchoffgrid](https://searchoffgrid.com/blog/ai-search-kpis-ecommerce)

6. **출력 포맷(metric_digest)**  
   - KPI 카드 리스트(값+편차+신호)  
   - 이상치 리스트(지표·가설·검증 액션)  
   - 추천 액션(직원별 핸드오프)  
   - 데이터 출처/기간/신뢰도  

나머지 길고 풍부한 예시는 SOP 문서에서 유지하면 좋겠습니다.

## 6. 분석가 직원 페이지 레이아웃 (`metric_digest`)

이제 분석가 직원 상세 페이지를,  
- “**경영자용 요약 → 운영자용 상세 → 가설/실험 리스트**”라는 3층 구조로 설계해 볼게요. [admetrics](https://www.admetrics.io/en/post/ai-powered-dashboards-dtc-ecommerce-71347)

### 6-1. 상단 헤더

**왼쪽**

- `📊 분석가 · KPI 집계·해석·이상치`  
- 한 줄 설명: `숫자를 통역해서, 어디가 문제인지·어디를 키워야 할지 제안하는 직원`  
- 현재 워크스페이스/브랜드명  

**오른쪽**

- 데이터 소스 뱃지:  
  - `스마트스토어`, `광고관리자`, `GA4`, (향후) `SNS 인사이트` 등  
- 모델 뱃지: `Haiku (집계) + Claude Sonnet (해석·가설)` [admetrics](https://www.admetrics.io/en/post/ai-powered-dashboards-dtc-ecommerce-71347)
- 상태: `활성 / 일시정지`  
- [📝 프롬프트] [⚙️ KPI·목표 설정] [🗑 해고] 버튼  

바로 아래 **요약 배너**:

- `이번 주: 매출 +12% 🟢 · 전환율 -0.4%p 🟠 · CAC 2배 🔴`  
- “데이터 기간: 6/8~6/14 · 업데이트 시각: 09:00”  

***

### 6-2. 기본 탭 구조

상단에 탭을 두면 좋습니다.

- `[📌 개요] [📈 지표 상세] [🔍 가설·실험] [📚 히스토리]`  

각 탭별로:

***

### 6-3. 개요 탭 — “사장이 30초 만에 보는 화면”

#### (A) KPI 카드 행

위쪽에 **KPI 카드 1~2줄**:

- 카드 예:  
  - `매출 320만 (전주 +12% · 🟢)`  
  - `주문수 80건 (+10% · 🟢)`  
  - `AOV 40,000원 (+2% · 🟢)`  
  - `전환율 2.1% (-0.4%p · 🟠)`  
  - `방문 1.5만 (+18% · 🟢)`  
  - `ROAS 4.8 (목표 3.0 이상 · 🟢)`  
  - `CAC 18,000원 (2배 ↑ · 🔴)`  

각 카드에는:

- 값  
- 기본 비교(전주/전월/목표 중 하나)  
- 색상/아이콘(🟢/🟠/🔴)  
- 작은 스파크라인(최근 4주 추세) [improvado](https://improvado.io/blog/web-analytics-dashboard)

#### (B) 이상치 리스트

KPI 카드 아래에 “이상치” 블록:

- 제목: `[이상치]`  
- 리스트:

  - `🔴 CAC 18,000원 (전월 9,000원, 2배)`  
    - `가설: 신규 광고 세트 효율 저조, 콜드 트래픽 비중 증가 (가설, 신뢰도 중간)`  
    - `검증: 세트·채널별 CAC 브레이크다운 확인 → 할일 생성`  
  - `🟠 전환율 2.5%→2.1% 하락`  
    - `가설: 인스타 유입 +18%, 상세 이탈률 +6% → 상세 첫 화면 메시지 미스매치 (가설)`  
    - `검증: 상세 첫 섹션 A/B 테스트(관리 쉬움 vs 가격)`  

각 항목 오른쪽에:

- `[할일 만들기]` – tasks에 바로 등록  
- `[광고에 넘기기]` / `[상세페이지에 넘기기]` – 관련 직원에게 핸드오프  

→ 이 블록만 봐도 “오늘/이번 주 어디가 문제인지”가 파악됩니다. [digitalapplied](https://www.digitalapplied.com/blog/ecommerce-analytics-kpis-dashboard-guide-2026)

#### (C) 추천 액션 섹션

이상치 리스트 아래:

- 제목: `[추천 액션]`  
- 정리된 2~4개 액션:

  1. `상세페이지 직원: 첫 섹션 A/B (관리 쉬움 vs 가격)`  
  2. `광고 직원: 효율 낮은 콜드 타겟 세트 예산 축소, 기능형 세트에 집중`  
  3. `SNS 직원: ‘관리 쉬운 통원목’ 교육형 콘텐츠 2개 추가`  

각 줄 옆에 `[→ 할일]` 또는 해당 직원 페이지로 이동하는 버튼.

***

### 6-4. 지표 상세 탭 — “운영자/분석자용”

여기서는 **KPI 묶음별로 표와 차트를 배치**합니다. [dataslayer](https://www.dataslayer.ai/blog/marketing-dashboard-best-practices-2025)

#### (A) 상단 필터

- 기간: `7일 / 30일 / 사용자 지정`  
- 세그먼트: `채널(검색/광고/SNS/직접)`, `디바이스(모바일/PC)` 등 (추후 확장). [portermetrics](https://portermetrics.com/en/compare/best-ecommerce-dashboard-templates/)

#### (B) 묶음별 섹션

1. **매출 묶음**

   - 표: `날짜 · 매출 · 주문수 · AOV`  
   - 라인차트: 매출/주문/AOV 추이.  

2. **유입 묶음**

   - 표: `채널 · 세션수 · 전환율 · ROAS · CAC`  
   - 바차트: 채널별 세션·전환율. [improvado](https://improvado.io/blog/web-analytics-dashboard)

3. **전환 묶음**

   - 퍼널: `방문 → 상품뷰 → 장바구니 → 주문` 단계별 전환. [portermetrics](https://portermetrics.com/en/compare/best-ecommerce-dashboard-templates/)
   - 이탈 구간 강조.  

4. **효율 묶음**

   - 표: `캠페인/세트 · CTR · CPC · CVR · CAC · ROAS` (광고에서 넘어온 데이터 포함). [improvado](https://improvado.io/blog/12-best-marketing-dashboard-examples-and-templates)

5. **재고 묶음(있는 경우)**

   - `상품 · 재고수 · 일 판매량 · 재고일수` 등.  

→ 이 탭은 “보고서다운 상세 뷰”라서, 초기에 일부만 구현하고 점차 확장해도 됩니다.

***

### 6-5. 가설·실험 탭 — “분석가의 메모장”

이 탭은 **이상치 → 가설 → 실험/액션**만 모아서 보는 곳입니다. [youtube](https://www.youtube.com/watch?v=Q3Pfe1VYRrU)

- 표:

  | 날짜 | 지표 | 신호 | 가설 요약 | 검증/실험 | 상태 |
  |------|------|------|-----------|-----------|------|
  | 6/8~6/14 | 전환율 | 🟠 | 상세 첫 화면 메시지 미스매치 | A/B 테스트 | 진행 중 |
  | 6/8~6/14 | CAC | 🔴 | 콜드 타겟 비중 증가 | 세트별 CAC 분석 | 할일 생성 |

- 각 행 클릭 시 오른쪽 패널에 상세 보기:

  - 이상치 상세(숫자·그래프)  
  - 가설(텍스트)  
  - “검증 방법” 설명  
  - 관련 직원(광고/상세/SNS) 링크  
  - 연결된 할일 상태  

→ 이 탭은 “분석가의 to-think 리스트”이고, **숫자에서 나온 가설/실험의 역사**를 남기는 역할을 합니다. [metricmosaic](https://www.metricmosaic.io/blog/website-analytics-dashboard)

***

### 6-6. 히스토리 탭 — 주간 metric_digest 아카이브

- 테이블:

  - `주차 · 매출 증감 · 전환율 증감 · 주요 신호(🔴/🟠 수) · 핵심 요약 한 줄`  
- 행 클릭 시 해당 주의 `metric_digest`(개요 탭)가 로딩.  

→ “지난 4주 동안 어떤 실험을 했고, KPI가 어떻게 움직였는지”를 돌아보는 데 유용합니다. [digitalapplied](https://www.digitalapplied.com/blog/ecommerce-analytics-kpis-dashboard-guide-2026)

***

### 6-7. “직접 시키기” 패널 (수동 분석 요청)

다른 직원처럼, 상단 또는 우측에 **사장 수동 입력 패널**을 둡니다.

- 타이틀: `직접 시키기 (수동 KPI 해석)`  
- 입력:

  - 지표 데이터 붙여넣기(textarea) — CSV/표 형태  
  - 기간 설명(예: “6/1~6/7 vs 5/25~5/31”)  
  - 궁금한 점: `예: 왜 ROAS는 올랐는데 매출이 정체인지 분석해줘`  

- [분석 실행] 버튼  

실행 시:

- Haiku가 붙여넣은 데이터를 간단히 파싱·집계.  
- Sonnet이 SOP에 맞춰  
  - KPI 카드  
  - 이상치  
  - 가설  
  - 추천 액션  
  구조로 metric_digest를 만들어 우측에 표시.  

→ 실데이터 연동 전에도, 사장이 스마트스토어/광고 관리자에서 숫자를 복붙해 빠르게 해석을 받는 용도로 상당히 유용합니다.

## 7. 정리

- 분석가 SOP는 **철학(숫자 통역사), KPI 묶음, 추이+이상치(신호등), 원인 가설→액션, 함정(상관≠인과, 표본·출처)** 측면에서, 현재 e커머스/마케팅 KPI 대시보드 가이드와 강하게 정렬되어 있습니다. [1clickreport](https://www.1clickreport.com/blog/marketing-kpi-dashboard-guide-2026)
- BASE_SOP.analyst에는  
  - 역할·철학  
  - KPI 묶음(8개 이내)  
  - 추이·이상치 규칙  
  - 가설·액션 구조  
  - 함정 주의  
  - metric_digest 포맷  
  정도만 압축해서 넣고, 지금 문서의 예시는 별도로 유지하면 모델 일관성이 좋아집니다. [1clickreport](https://www.1clickreport.com/blog/marketing-kpi-dashboard-guide-2026)
- 분석가 직원 페이지는  
  - 개요 탭: KPI 카드 + 이상치 리스트 + 추천 액션(사장 30초용)  
  - 지표 상세 탭: 묶음별 표·차트(운영자·분석자용)  
  - 가설·실험 탭: 이상치→가설→실험/할일의 역사  
  - 히스토리 탭: 주간 metric_digest 아카이브  
  - 공통: 상단 “직접 시키기” 패널  
  구조로 설계하면, 실제로 “숫자를 모으는 대시보드”가 아니라 **숫자를 해석하고 행동으로 연결해주는 ‘분석가 한 명’을 그대로 시각화한 화면**이 됩니다. [dataslayer](https://www.dataslayer.ai/blog/marketing-dashboard-best-practices-2025)
