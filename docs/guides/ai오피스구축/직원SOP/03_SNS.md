## 1. 철학 — "팔려고 들이대지 말고, 멈추게 하고 머물게 하라."

SNS는 광고판이 아니라 **관계의 장**이다. 판매글만 올리면 사람은 떠난다. 먼저 **스크롤을 멈추게**(첫 줄) 하고, **머물게**(공감·가치) 한 뒤, 제품은 **자연스럽게** 흘린다. 좋아요가 목적이 아니라 **저장·공유·문의**로 이어지는 콘텐츠를 만든다.

---

## 2. 콘텐츠 목적 + 훅 공식 (검증 반영)

### 2-1. 먼저 "목적"을 정한다 (유형보다 목적이 먼저 — GPT 검증)
같은 도마 콘텐츠도 목적에 따라 다르게 만든다.
| 목적 | 노림수 | 예 |
|---|---|---|
| 인지 | 브랜드 첫인상 | "주방 분위기를 바꾸는 작은 기준" |
| 저장 | 유용해서 저장 | "원목 도마 오래 쓰는 3가지 관리법" |
| 공감 | 관계 형성 | "신혼집 주방 꾸밀 때 은근 어려운 것" |
| 문의 | 댓글·DM 유도 | "사이즈 고민되면 댓글로" |
| 전환 | 구매 이동 | "이번 주 신상 도마 오픈" |
| 신뢰 | 안심 | "실제 고객 주방에 놓인 모습" |

### 2-2. 훅 8유형 + 훅 점수
캡션 첫 줄은 8유형 중 하나로 스크롤을 멈춘다. **훅 후보 2~3개를 만들어 A/B에 분배**(퍼플렉시티).
| 유형 | 예시 (시목 도마) |
|---|---|
| 질문 | "신혼집 주방, 딱 하나만 둔다면?" |
| 공감 | "플라스틱 도마 칼자국, 좀 찝찝하셨죠" |
| 반전 | "비싼 도마가 아니라, 오래 쓰는 도마예요" |
| 숫자 | "10년 쓰는 도마의 3가지 조건" |
| 도발 | "원목 도마, 아직 그냥 사세요?" |
| 상황 | "집들이 선물 고르다 다 비슷해 보인다면" |
| 체크리스트 | "원목 도마 사기 전, 이 3가지만 보세요" |
| 비교 | "플라스틱 vs 원목 도마, 차이는 여기서" |

**훅 점수**(예: 8/10): 즉시성·타겟성·저장성·전환성·브랜드 적합성 5기준 자가 평가.
> 첫 줄 다음: **공감 → 가치 → 자연스러운 제품 언급** → 해시태그 → 목적에 맞는 CTA.

## 3. 콘텐츠 캘린더 + 믹스 비율 (캠페인 모드별)

**판매글만 올리지 않는다.** 평상시 기본 믹스는 정보40·공감20·제품20·후기10·비하인드10.

**캠페인 모드(campaign_mode)에 따라 비율이 달라진다** (GPT 검증):
| 모드 | 핵심 조정 |
|---|---|
| 평상시 | 정보40·공감20·제품20·후기10·비하인드10 (기본) |
| 초기 브랜드 | 비하인드↑(15) — "왜 파는지/누가 운영하는지"가 후기보다 먼저 |
| 판매집중(신상·공구) | 제품35·후기20·정보25 |
| 신뢰회복·브랜딩 | 비하인드25·후기20·제품10 |

> 각 게시물: **날짜 · 채널 · 포맷 · 유형 · 목적 · 주제 · 핵심 메시지 · 해시태그**. (content_pillar + 캠페인 태그를 달아 분석가가 성과 분석)

## 4. 변주 · 플랫폼 · 릴스 · 해시태그 · CTA (검증 반영)

### 4-1. A/B/C 변주 — 한 번에 한 변수만
A(감성)/B(정보)/C(제품) 각도. **테스트는 한 번에 하나의 변수만**(훅 vs 본문 vs 썸네일) — 분석가가 성과를 명확히 해석하게.

### 4-2. 플랫폼 × 포맷 (결과물이 달라야 함)
| 플랫폼·포맷 | 핵심 |
|---|---|
| 인스타 피드 | 4:5, 첫 줄 훅, 저장/문의 유도 |
| 인스타 릴스 / 쇼츠 | 0~3초 장면, 자막, 컷 구성, 오디오 무드 |
| 인스타 스토리 | 투표·질문·링크·비하인드 |
| 스레드 | 짧은 생각·공감·대화 유도 |
| 블로그 | 검색형 제목·목차·정보 본문 |
> 입력은 **플랫폼 × 포맷 × 목적**으로 분리.

### 4-3. 릴스 타임라인 (장면 구성이 핵심)
`0~3초 후킹 → 3~8초 문제 → 8~18초 해결 → 18~25초 제품 자연 노출 → 25~30초 저장/문의 CTA`.
각 컷: **장면 · 자막 · 촬영가이드 · 소품 · 컷길이 · CTA**.

### 4-4. 해시태그 역할별 (그냥 많이 X)
대형 2(노출) + 중형 4~5(타겟) + 소형 4~5(전환) + **브랜드 1~2(누적, #시목)**.
> 건강식품은 해시태그도 컴플라이언스 — #면역력개선·#다이어트효과 등 금지.

### 4-5. CTA는 목적에 맞춰 자동 선택
저장 목적→저장 CTA / 문의→댓글·DM / 전환→링크 / 신뢰→경험 공유. (좋아요만 보지 말고 **저장·공유·문의** KPI 중시)

## 5. 출력 구조 (sns_queue)

```
📅 6/16(월) 09:00 · 인스타 피드 · [제품]
  훅(질문): "신혼집 주방, 딱 하나만 둔다면?"
  본문: 통원목 티크 도마 + 신혼 선물 각도 (공감→가치→제품)
  #신혼선물 #원목도마 #주방인테리어 (대형2+중형5+소형5)
  🖼 이미지 브리프: 자연광·원목 식탁·미니멀 (→ 비주얼 디렉터)
  [A 감성 / B 정보 / C 제품] [글자수 ✅] [승인→예약] [복사]
[넘기기] 반응 좋은 콘텐츠 → 광고 직원 소재 / 긍정 댓글 → CS
```
> **저장은 JSON 병행**(MVP-2): `{ posts:[{ date, channel, type, hook, body, hashtags, imageBrief, variants:[A,B,C], status }] }`

# GPT 검증

좋아요. SNS 운영자 SOP는 전체 방향이 좋습니다.
특히 “판매글만 올리지 않는다”, “첫 줄 훅”, “콘텐츠 믹스 비율”, **“A/B/C 변주”**까지 들어간 점이 실무적으로 아주 좋아요.

다만 이 직원은 앞의 소싱기획자·CS 매니저와 달리 산출물이 가장 많이 쌓이는 직원입니다.
그래서 단순히 캡션을 잘 쓰는 것보다, 아래 4가지가 더 중요합니다.

1. 콘텐츠를 캘린더에서 관리할 수 있는가
2. 승인/예약/발행 상태가 보이는가
3. 플랫폼별 형식이 분리되는가
4. 반응 좋은 콘텐츠가 광고·상세페이지·CS로 재활용되는가

한 줄로 정리하면:

SNS 운영자는 “캡션 작성봇”이 아니라 “콘텐츠 캘린더 매니저 + 브랜드 말투 관리자 + 반응 데이터 전달자”로 설계해야 합니다.

⸻

1. SOP 전체 점검

좋은 점

현재 SOP의 방향은 상당히 좋습니다.

특히 이 문장:

SNS는 광고판이 아니라 관계의 장이다.

이건 SNS 직원의 핵심 정체성으로 아주 적합합니다.

SNS 직원이 매번 제품만 팔려고 하면 브랜드 피드가 금방 피로해집니다.
그래서 정보, 공감, 제품, 후기, 비하인드를 섞는 방식은 꼭 유지해야 합니다.

현재 구조에서 특히 좋은 요소는 아래입니다.

항목	평가
철학	좋음. 판매보다 관계 중심
훅 공식	좋음. 첫 줄을 명확히 제어함
콘텐츠 믹스	좋음. 피드 피로도 방지
A/B/C 변주	좋음. 테스트 가능
릴스 구조	방향 좋음. 더 구체화 필요
넘기기 구조	좋음. 광고/비주얼/CS와 연결 가능

⸻

2. 가장 크게 보완해야 할 점

보완 1. SNS 직원은 “목적”을 먼저 분류해야 함

현재는 콘텐츠 유형은 있는데, 콘텐츠 목적이 부족합니다.

SNS 콘텐츠는 유형보다 목적이 먼저입니다.

예를 들어 같은 “원목 도마” 콘텐츠라도 목적이 다를 수 있습니다.

인지: 시목이라는 브랜드를 처음 알게 만들기
저장: 원목 도마 관리법을 저장하게 만들기
문의: 가격/구매처를 물어보게 만들기
전환: 스마트스토어로 이동하게 만들기
신뢰: 후기/제작과정으로 안심시키기

그래서 SOP에 아래 목적 분류를 추가하는 게 좋습니다.

목적	설명	예
인지	브랜드/제품을 처음 알림	“주방 분위기를 바꾸는 작은 기준”
저장	유용한 정보 제공	“원목 도마 오래 쓰는 3가지 관리법”
공감	관계 형성	“신혼집 주방 꾸밀 때 은근 어려운 것”
문의	댓글/DM 유도	“도마 사이즈 고민되면 댓글로 알려주세요”
전환	구매 유도	“이번 주 신상 도마 오픈”
신뢰	후기/제작과정	“실제 고객 주방에 놓인 모습”

콘텐츠 캘린더에는 유형뿐 아니라 목적이 같이 들어가야 합니다.

날짜 · 채널 · 유형 · 목적 · 주제 · 핵심 메시지 · CTA

⸻

보완 2. 콘텐츠 믹스 비율은 브랜드 단계별로 달라야 함

현재 비율:

정보·팁 40%
공감·일상 20%
제품 20%
후기·UGC 10%
비하인드 10%

이 비율은 안정적입니다.
다만 브랜드 상황에 따라 달라져야 합니다.

아직 팔로워가 적고 신뢰가 없는 상태.

정보·팁 35%
공감·일상 20%
제품 20%
비하인드 15%

신상 오픈, 공동구매, 이벤트 기간.

정보·팁 25%

클레임 이후, 리브랜딩, 가격 인상 전후.

정보·팁 30%

후기·UGC 20%

보완 3. 플랫폼별 포맷 차이를 더 강하게 넣어야 함

현재 인스타, 유튜브, 블로그가 언급되어 있는데, 실제 출력은 인스타 중심입니다.

SNS 운영자가 진짜 실무적이려면 플랫폼별 결과물이 달라야 합니다.

플랫폼	핵심
인스타 피드	4:5 이미지, 첫 줄 훅, 저장/문의 유도
인스타 릴스	0~3초 장면, 자막, 컷 구성, 오디오 무드
인스타 스토리	투표/질문/링크/비하인드
스레드	짧은 생각, 공감형, 대화 유도
블로그	검색형 제목, 목차, 정보성 본문
유튜브 쇼츠	장면 전환, 자막, 15~30초 유지율

따라서 입력 폼에도 단순 플랫폼만 두지 말고, 아래처럼 분리하는 게 좋습니다.

플랫폼: 인스타 / 스레드 / 블로그 / 유튜브
포맷: 피드 / 릴스 / 스토리 / 쇼츠 / 글
목적: 인지 / 저장 / 문의 / 전환 / 신뢰

⸻

보완 4. “성과 반영 루프”가 필요함

SNS 직원은 콘텐츠를 만들고 끝나면 안 됩니다.

반드시 아래 루프가 있어야 합니다.

게시물 생성
→ 승인/예약
→ 발행
→ 반응 기록
→ 잘 된 이유 분석
→ 다음 콘텐츠에 반영
→ 광고 직원에게 소재 전달

즉, SNS 직원의 진짜 성장 포인트는 성과 데이터 학습입니다.

처음에는 수동 입력으로도 충분합니다.

좋아요 수
저장 수
공유 수
댓글 수
문의 수
프로필 방문
링크 클릭

이 중 SNS에서는 특히 저장/공유/문의를 중요하게 봐야 합니다.
좋아요만 보면 예쁜 콘텐츠만 늘고 매출과 멀어질 수 있습니다.

3. 훅 공식 보완

현재 훅 5유형은 좋습니다.

질문 / 공감 / 반전 / 숫자 / 도발

여기에 3개를 추가하면 더 좋습니다.

유형	예시
상황	“집들이 선물 고르다 결국 다 비슷해 보인다면”
체크리스트	“원목 도마 사기 전, 이 3가지만 보세요”

최종 훅 유형은 8개 추천입니다.

질문 / 공감 / 반전 / 숫자 / 도발 / 상황 / 체크리스트 / 비교

훅 점수도 추가하면 좋음

AI가 만든 훅마다 점수를 매기게 하면 실무적으로 좋습니다.

훅 점수: 8/10
이유: 타겟의 불편을 직접 건드리고 저장 가능성이 있음

평가 기준:

기준	설명
즉시성	첫 줄만 봐도 멈출 만한가
타겟성	우리 고객이 자기 얘기로 느끼는가
저장성	나중에 다시 보고 싶은가
전환성	제품/문의로 자연스럽게 이어지는가
브랜드 적합성	과장·자극 없이 브랜드 톤과 맞는가

4. 릴스 SOP 보완

릴스는 지금보다 더 구체화해야 합니다.

릴스는 캡션보다 장면 구성이 핵심입니다.

추천 구조:

0~3초: 후킹 장면
3~8초: 문제/상황 제시
8~18초: 해결/정보
18~25초: 제품 자연 노출
25~30초: 저장/문의 CTA

릴스 출력 예시

릴스 30초 구성
0~3초
장면: 칼자국 많은 플라스틱 도마 클로즈업
자막: “이 칼자국, 매일 보이셨죠?”
3~8초
장면: 원목 도마를 꺼내는 손
자막: “주방 분위기는 도마 하나로도 달라져요”
8~18초
장면: 도마 위에 과일/빵 플레이팅
자막: “티크는 단단하고 수분에 강한 원목이에요”
18~25초
장면: 식탁 위 전체 무드샷
자막: “오래 쓰는 물건을 고르고 싶다면”
25~30초
장면: 제품 단독 컷
자막: “저장해두고 도마 고를 때 확인하세요”

릴스에는 반드시 아래 필드가 있어야 합니다.

장면 / 자막 / 촬영 가이드 / 소품 / 컷 길이 / CTA

6. CTA 보완

현재 “부드러운 CTA”가 들어가 있는데, CTA 유형을 구체화하면 좋습니다.

CTA 유형	예
저장	“도마 고르기 전 저장해두세요.”
댓글	“주방에서 제일 자주 쓰는 도구는 뭐예요?”
DM	“사이즈 고민되면 DM으로 주방 사진 보내주세요.”
링크	“구매는 프로필 링크에서 확인하실 수 있어요.”
공유	“신혼집 준비 중인 친구에게 보내주세요.”
문의	“선물용 구성은 댓글로 문의주세요.”

SNS 직원은 게시물 목적에 따라 CTA를 자동 선택해야 합니다.

저장 목적 → 저장 CTA
문의 목적 → 댓글/DM CTA
전환 목적 → 링크 CTA
신뢰 목적 → 후기/경험 공유 CTA

8. SNS 운영자 직원 페이지 방향

SNS 직원 페이지는 캘린더 + 콘텐츠 큐가 중심이어야 합니다.

소싱은 분석 리포트 중심,
CS는 티켓 큐 중심,
SNS는 콘텐츠 일정과 상태 관리가 중심입니다.

이 직원 페이지에 들어갔을 때 사장이 바로 봐야 하는 것은 아래입니다.

이번 주에 뭐가 올라가나?
승인해야 할 콘텐츠가 몇 개인가?
어떤 콘텐츠가 피드를 너무 판매글처럼 만들고 있나?
릴스/피드/스토리 비율이 맞나?
비주얼 디렉터에게 넘길 이미지 브리프가 있나?

22. 다른 직원과의 연결

SNS 운영자는 다른 직원들의 결과물을 받아서 콘텐츠화하는 직원입니다.

연결 직원	SNS가 받는 것	SNS가 만드는 것
소싱기획자	타겟, USP, 구매장벽	제품 소개 피드, 신상 런칭 콘텐츠
CS 매니저	반복 문의, 긍정 후기	FAQ 콘텐츠, 후기 콘텐츠
모니터링	트렌드 키워드, 경쟁사 흐름	트렌드형 콘텐츠
비주얼 디렉터	촬영컷, 이미지 프롬프트	발행 가능한 콘텐츠 세트
광고 기획자	반응 좋은 카피/소구점	광고 전 테스트 콘텐츠
분석가	성과 좋은 콘텐츠 유형	다음 캘린더 조정

SNS가 다른 직원에게 넘기는 것:

넘길 직원	넘길 내용
비주얼 디렉터	이미지/릴스 촬영 브리프
광고 기획자	반응 좋은 훅, 저장 많은 콘텐츠
CS 매니저	긍정 댓글, 부정 댓글, 문의 유입
상세페이지 제작자	저장 많은 정보형 콘텐츠, 고객 반응 문구
분석가	콘텐츠 성과 데이터

⸻

23. SNS 직원에서 특히 조심할 점

1) 캘린더가 복잡해지면 사장이 안 씀

너무 많은 필터와 기능을 한 화면에 넣으면 복잡해집니다.
그래서 기본 화면은 아주 단순해야 합니다.

오늘 / 이번 주 / 승인대기 / 비주얼필요

이 4개만 먼저 보여줘도 충분합니다.

⸻

2) 자동 발행은 나중에

MVP에서는 자동 발행까지 가지 않는 게 좋습니다.

처음에는:

AI 생성 → 사장 승인 → 복사 또는 예약완료 체크

그다음 확장:

Meta API / YouTube API / 블로그 연동 → 실제 예약 발행

초기부터 자동 발행을 붙이면 OAuth, 권한, 실패 처리, 이미지 업로드, 글자수 제한 등으로 제품 복잡도가 확 올라갑니다.

⸻

3) 브랜드 톤 점검 버튼이 필요함

SNS는 말투가 가장 중요합니다.

각 콘텐츠 상세에 아래 버튼을 추천합니다.

[브랜드 톤 점검]
[더 담백하게]
[더 감성적으로]
[판매 느낌 줄이기]
[짧게 줄이기]

특히 “판매 느낌 줄이기” 버튼은 시목 같은 브랜드에 아주 유용합니다.

⸻

4) 건강식품 워크스페이스는 컴플라이언스 체크 필수

SNS는 표현이 짧고 자극적으로 가기 쉬워서 건강식품에서 위험합니다.

예를 들어:

피로 회복
면역력 개선
살 빠지는
혈당 잡는
염증 줄이는

이런 표현은 위험할 수 있습니다.

따라서 쏠닝포인트 같은 건강식품 브랜드에서는 콘텐츠 생성 후 반드시:

[금지표현 검사]

버튼 또는 자동 체크가 있어야 합니다.

⸻

24. 최종 판정

SNS 운영자 SOP는 기본 방향 합격입니다.
다만 실무 제품으로 만들려면 아래를 꼭 반영하는 게 좋습니다.

반드시 수정

콘텐츠 목적 추가: 인지/저장/공감/문의/전환/신뢰
캠페인 모드 추가: 평상시/신상런칭/이벤트/브랜딩/판매집중
플랫폼별 포맷 분리
릴스 장면 구조 강화
성과 반영 루프 추가
콘텐츠 상태값 추가

페이지에서 반드시 보여줄 것

1. 이번 주 콘텐츠 수
2. 승인대기 콘텐츠
3. 비주얼 필요 콘텐츠
4. 콘텐츠 믹스 비율
5. 캘린더
6. 콘텐츠 카드
7. A/B/C 변주
8. 이미지/릴스 브리프
9. 다른 직원에게 넘기기 버튼
10. 성과/재활용 후보

최종 한 줄

SNS 운영자는 “오늘 올릴 글을 써주는 직원”이 아니라, 브랜드의 콘텐츠 흐름을 관리하고 반응 좋은 소재를 회사 전체 자산으로 바꾸는 직원이어야 합니다.

이 직원의 핵심 화면은 결국 이 흐름이어야 합니다.

아이디어 수집
→ 콘텐츠 생성
→ 캘린더 배치
→ 비주얼 요청
→ 승인
→ 발행
→ 성과 기록
→ 재활용/광고 연결

이 흐름까지 잡히면 SNS 운영자는 단순한 캡션봇이 아니라, 진짜로 회사 안에서 매일 콘텐츠를 굴리는 직원처럼 느껴질 거예요.

# 퍼플렉시티 검증

SNS 운영자 SOP 자체는 “훅 공식 + 캘린더 + A/B 변주”를 잘 잡은 상태라, 큰 틀은 그대로 쓰셔도 됩니다. [genailast](https://www.genailast.com/blogs/how-to-use-ai-for-social-media-content-creation.php)
여기서는 (1) SOP 내용 디테일 보완, (2) BASE_SOP.sns에 어떤 식으로 압축할지, (3) 직원 페이지 레이아웃을 **실제 작업 흐름 기준으로** 구체화해서 정리하겠습니다. [kirkgroup](https://kirkgroup.com/blog/creating-social-media-content-calendars-with-ai/)

***

## 1. SNS SOP 내용 점검 (철학·훅·캘린더·변주)

### 1-1. 철학 — “멈추게 하고 머물게 하라”

- “광고판이 아니라 관계의 장”, “스크롤 멈춤 → 머묾 → 자연스러운 제품 언급” 구조는 2026년 SNS 마케팅 가이드에서 말하는 **교육/공감/브랜드 중심 콘텐츠 비중을 늘려 전환으로 이어지게 하는 전략**과 잘 맞습니다. [midastools](https://www.midastools.co/blog/ai-social-media-prompts-2026)
- “좋아요가 목적이 아니라 저장·공유·문의”라는 KPI 정의도 요즘 IG/TikTok 운영에서 추천하는 **세이브/쉐어/프로필 방문/DM 위주의 지표 포커스**와 일치합니다. [postquick](https://www.postquick.ai/blog/ai-social-media)

→ 철학은 이미 잘 잡혀 있고, BASE_SOP에는 ‘**저장/공유/문의로 이어질 콘텐츠 우선**’이라고 명시해주는 정도면 충분합니다.

***

### 1-2. 훅 공식 — 5유형 첫 줄

질문·공감·반전·숫자·도발 5유형은 실제로 AI 소셜 캘린더 툴들이 추천하는 대표 훅 유형과 거의 동일합니다. [aiuxplayground](https://www.aiuxplayground.com/prompts/social-media-content)

**좋은 점**

- “훅 유형을 고정해두고, 그 안에서 변주”하는 방식이 캡션 A/B 테스트에 아주 잘 맞습니다. [mymarky](https://mymarky.com/blog/best-ai-social-media-ab-testing-tool-2026)
- 각 유형 예시에 시목 도마를 박아둔 것도, 브랜드별 few-shot으로 쓰기 좋습니다.  

**보완 제안**

- BASE_SOP에 “가능하면 **훅 2–3개를 생성**하고, 그중에서 1개를 선택(또는 A/B/C에 분배)”하도록 적어두면, 나중에 A/B 테스트와 자연스럽게 이어집니다. [midastools](https://www.midastools.co/blog/ai-social-media-prompts-2026)
- 훅 뒤 구조(공감→가치→자연스러운 제품 언급→해시태그→부드러운 CTA)를 지금처럼 명시해두면, 모델이 “팔이 안 꺾인 자연스러운 판매글”을 일관되게 생성하기 좋습니다. [happycapyguide](https://happycapyguide.com/blog/how-to-use-ai-for-social-media-2026)

***

### 1-3. 콘텐츠 캘린더·믹스 비율

정보/팁 40% · 공감/일상 20% · 제품 20% · 후기·UGC 10% · 비하인드 10%는, 외부 가이드들이 추천하는 **교육·정보성 40% 내외 + 프로모션 20% + 나머지 관계/브랜드** 구조와 거의 일치합니다. [genailast](https://www.genailast.com/blogs/how-to-use-ai-for-social-media-content-creation.php)

**딱 좋은 점**

- “유형”을 미리 정의하고 여기에 맞춰 캘린더를 채우는 건, “먼저 전략(캘린더)을 짜고 그 안을 AI로 채워라”는 최신 베스트프랙티스와 동일합니다. [apaya](https://apaya.com/blog/ai-content-calendar)

**추가로 넣으면 좋은 것**

- 캘린더 항목에 `content_pillar`(예: 정보/공감/제품/UGC/비하인드)와 **캠페인 태그**를 함께 넣으면, 나중에 분석가 직원이 “어떤 유형/캠페인이 저장·공유를 많이 일으켰는지”를 분석하기 쉬워집니다. [taskade](https://www.taskade.com/generate/social-media-calendar/multi-channel-social-calendar)

***

### 1-4. A/B/C 변주·플랫폼별 대응

- A/B/C를 감성/정보/제품 각도로 두는 건, 지금 당장 실험할 수 있는 좋은 축입니다. [mymarky](https://mymarky.com/blog/best-ai-social-media-ab-testing-tool-2026)
- 해시태그를 대형/중형/소형으로 나누는 것도, 실제 IG 해시태그 전략으로 자주 언급되는 방식입니다. [postquick](https://www.postquick.ai/blog/ai-social-media)
- 릴스는 0–3초 후킹 장면부터, 장면별 스크립트 + 자막 구조로 짠다는 것도 요즘 릴스/TikTok 운영 가이드와 일치합니다. [happycapyguide](https://happycapyguide.com/blog/how-to-use-ai-for-social-media-2026)

**보완하면 좋은 점**

- BASE_SOP에 “A/B 테스트는 **한 번에 하나의 변수만 바꿔라(훅 vs 본문 vs 썸네일)**” 정도를 명시하면, 나중에 분석가가 성과 해석할 때 혼선이 줄어듭니다. [kirkgroup](https://kirkgroup.com/blog/creating-social-media-content-calendars-with-ai/)
- 플랫폼별 특징(인스타=짧고 감성, 유튜브=제목+썸네일, 블로그=검색 키워드)을 아주 간단히라도 규칙으로 넣어두면, 멀티채널 대응 시 더 일관된 결과를 얻을 수 있습니다. [genailast](https://www.genailast.com/blogs/how-to-use-ai-for-social-media-content-creation.php)

***

### 1-5. 모델 파이프라인 (GPT 계열 → Claude 톤 정제)

- 외부 벤치마크 기준, GPT-4o/5 계열은 **속도·변주 폭·명령 순종성**이 좋아서 훅·캡션·스크립트를 대량으로 뽑는 데 적합합니다. [jeeva](https://www.jeeva.ai/blog/gpt4o-vs-claude-sonnet-sales-copy-benchmarks)
- Claude Sonnet은 **브랜드 톤 유지, 인간스러운 문장, 긴 프롬프트 준수**에서 강점을 보여서, “브랜드 톤 정제” 단계에 쓰기 좋습니다. [aloud](https://aloud.ie/blog/claude-vs-gpt4o-marketing-copy)

→ “변형 대량=GPT 계열, 최종 톤 정제=Claude Sonnet” 구조는 **실무 에이전트 파이프라인에서 추천되는 하이브리드 패턴 그대로**입니다. [adlibrary](https://adlibrary.com/posts/claude-vs-chatgpt-for-marketers)

***

### 1-6. BASE_SOP.sns에 꼭 들어갈 최소 핵심

1. **역할·철학 2–3줄**  
   - 스크롤 멈춤(훅) → 머묾(공감·가치) → 자연스러운 제품 언급 → 저장/공유/문의 KPI.  
2. **훅 규칙**  
   - 5유형 정의 + 가능하면 2–3개 훅 후보 생성 후, A/B에 분배.  
3. **콘텐츠 믹스 규칙**  
   - 유형별 비중(정보/공감/제품/UGC/비하인드)과 캘린더 필수 필드(날짜, 채널, 유형, 주제, 메시지 핵심, 해시태그, imageBrief).  
4. **A/B/C 변주 규칙**  
   - A=감성, B=정보, C=제품 각도, 한 번에 하나의 변수만 테스트.  
5. **출력 포맷(sns_queue)**  
   - 게시물 큐 레코드 구조(날짜, 채널, 유형, 훅, 본문, 해시태그, imageBrief, variants, status).  

나머지 긴 예시는 지금처럼 별 SOP 파일로 두면 충분합니다. [aiuxplayground](https://www.aiuxplayground.com/prompts/social-media-content)

***

## 2. SNS 직원 페이지 레이아웃 (`sns_queue`)

이제 **SNS 운영자 직원 상세 페이지**를, 실제 인스타/유튜브 운영 워크플로 기준으로 짜보겠습니다.  
키워드는 **캘린더 중심 + 큐 리스트 + 변주 관리 + 성과 피드백**입니다. [koala](https://koala.sh/tools/free-ai-social-media-calendar-generator)

### 2-1. 상단 헤더

**왼쪽**

- `📣 SNS 운영자 · 캘린더 + 캡션 + 릴스 스크립트`  
- 한 줄 설명: `팔려고 들이대지 않고, 멈추게 하고 머물게 하는 콘텐츠를 만드는 직원`  
- “브랜드” 표시: `시목 · 인스타/유튜브 중심`  

**오른쪽**

- 모델 뱃지: `GPT 계열 (아이데이션) + Claude Sonnet (톤 정제)`  
- 상태: `활성 / 일시정지` 토글  
- 버튼: `[📝 프롬프트] [⚙️ 캘린더 설정] [🗑 해고]`  

그 아래 **이번 주 요약 배너**:

- `이번 주 예정 7개 · 승인 완료 4 · 작성 중 3 · 반응 상위 포스트 1개 하이라이트`  
- (나중에 KPI 연동 시) `지난 7일 평균 저장 68건 / 공유 24건` 정도 표시. [postquick](https://www.postquick.ai/blog/ai-social-media)

***

### 2-2. 메인 뷰 – “캘린더 + 큐” 2단 구조

가장 추천하는 구조는, 화면을 **좌: 캘린더 / 우: 선택된 날짜의 큐 리스트+세부**로 나누는 것입니다.

#### (A) 좌측 – 캘린더

- 월/주 뷰 전환: `월간 캘린더` 기본, `주간 리스트` 옵션.  
- 각 날짜 칸에:  
  - 플랫폼 아이콘(IG, YT, Blog)  
  - “포스트 개수 배지” (예: 2, 3)  
  - 간단한 `유형` 아이콘 (정보/공감/제품/UGC/비하인드).  

날짜를 클릭하면 오른쪽 큐가 해당 날짜의 게시물들로 필터링됩니다.

#### (B) 우측 상단 – 큐 리스트 (sns_queue)

- 필터 바:  
  - `채널(인스타/유튜브/블로그)`  
  - `유형(정보/공감/제품/UGC/비하인드)`  
  - `상태(초안/승인대기/예약완료/발행완료)`  

- 큐 카드 예시 (각 포스트):  

  - 첫 줄:  
    - `📅 6/16(월) 09:00 · 인스타 피드 · [제품] · 상태: 승인 대기`  
  - 두 번째 줄:  
    - `훅 유형(질문/공감/반전/숫자/도발)` + 훅 텍스트 첫 줄  
  - 메타:  
    - `A/B/C 활성 여부`, `imageBrief 요약(자연광·4:5)`  
  - 액션 버튼:  
    - `[편집] [복사] [승인→예약]`  

큐에서 하나를 클릭하면, 우측 하단 혹은 오른쪽 패널에 상세 에디터가 열립니다.

***

### 2-3. 우측 하단/패널 – 게시물 상세 에디터

**1) 상단 메타**

- `날짜/시간` (datetime picker)  
- `채널` (인스타 피드/릴스/스토리, 유튜브, 블로그…)  
- `유형` (정보/공감/제품/UGC/비하인드)  
- 상태 드롭다운: `초안 / 승인 대기 / 예약 / 발행 완료`  

**2) 콘텐츠 영역**

- **훅 영역**

  - `훅 유형` (질문/공감/반전/숫자/도발) select  
  - 텍스트 입력: 첫 줄  
  - A/B/C용 훅 3개를 탭으로 전환해서 볼 수 있게:  
    - `[A 감성] [B 정보] [C 제품]` 탭  

- **본문 캡션**

  - 각 탭별 캡션 텍스트 영역:  
    - A: 감성 버전  
    - B: 정보 버전  
    - C: 제품 소개 버전  
  - `글자수 카운터` + `플랫폼 규정 체크(IG 최대 글자수 등)` 표시. [genailast](https://www.genailast.com/blogs/how-to-use-ai-for-social-media-content-creation.php)

- **해시태그·CTA**

  - 해시태그 입력 + `대형/중형/소형` 태그 칩 색상 구분.  
  - 부드러운 CTA 문장 필드 (예: “댓글로 ○○ 알려주세요”, “프로필 링크에서 자세히 보기”).  

- **이미지 브리프**

  - `imageBrief` 텍스트 + 태그:  
    - 예: `자연광, 원목 식탁, 미니멀, 4:5`  
  - `→ 비주얼 디렉터에게 보내기` 버튼 (클릭 시 visual 직원의 “직접 시키기” 폼으로 해당 브리프 전송).  

**3) 릴스 스크립트 섹션 (릴스/영상일 때만)**

- `총 길이 선택: 15/30/60초`  
- 장면 리스트:  
  - `0–3초: 후킹 장면 + 내레이션 (또는 자막)`  
  - `3–10초: 메인 메시지`  
  - `10–15초: 제품/CTA`  
- 각 장면마다:  
  - 텍스트 스크립트 + “자막 문장” + “샷 아이디어”(예: 손이 도마를 쓰는 장면).  

→ 이 에디터 하나로 **텍스트·훅·변주·이미지 브리프·릴스 스크립트까지 한 번에 관리**할 수 있습니다. [midastools](https://www.midastools.co/blog/ai-social-media-prompts-2026)

***

### 2-4. “직접 시키기” 패널 (사장 입력 → 즉시 생성)

다른 직원과 동일하게, SNS도 상단이나 우측에 **“직접 시키기”** 박스를 두면 좋습니다.

- 타이틀: `직접 시키기 (개별 게시물 생성)`  
- 입력 필드:  
  - 주제/상품  
  - 채널 (인스타 피드/릴스/스토리/유튜브/블로그 등)  
  - 길이(15/30/60초 or 글자수 짧게/보통/길게)  
  - 콘텐츠 유형(정보/공감/제품/후기/비하인드)  

- [캡션 + 훅 + 해시태그 + 이미지 브리프 생성] 버튼  

실행 결과:

- GPT 계열이 아이디어/변주 1차 생성 → Claude가 톤 정제 → 바로 아래에 `sns_queue` 카드 1개(또는 여러 개)로 추가. [adlibrary](https://adlibrary.com/posts/claude-vs-chatgpt-for-marketers)

***

### 2-5. 성과/인사이트 연동 영역

SNS 직원의 강점은 **성과를 즉시 다음 캘린더에 반영**하는 것입니다. [kirkgroup](https://kirkgroup.com/blog/creating-social-media-content-calendars-with-ai/)

페이지 하단에 작은 “성과/인사이트” 섹션을 두면 좋습니다.

- `최근 7일 상위 3개 포스트` 카드:  
  - 저장/공유/댓글 수  
  - 어떤 유형·훅·콘텐츠 축이었는지  
- `추천 액션`:  
  - “정보형 B 변주가 가장 반응이 좋으니 다음 주 같은 포맷으로 2개 더 제안”  
  - “저장 많은 포스트를 광고 직원에게 넘겨, 소재로 재활용하자” [mymarky](https://mymarky.com/blog/best-ai-social-media-ab-testing-tool-2026)

각 카드에:

- `[광고 직원에게 소재로 넘기기]`  
- `[콘텐츠 복제 → 새 날짜에 배치]`  

이런 CTA를 붙이면, SNS↔광고↔분석가 협업 루프가 자연스럽게 이어집니다.

## 3. 한 줄 정리

- SNS SOP의 철학(관계, 멈춤→머묾→자연스러운 제품 언급), 훅 공식, 콘텐츠 믹스, A/B/C 변주 구조는 **현재 AI 기반 SNS 운영 베스트프랙티스와 잘 정렬**되어 있습니다. [happycapyguide](https://happycapyguide.com/blog/how-to-use-ai-for-social-media-2026)
- BASE_SOP.sns에는 역할·훅 규칙·콘텐츠 믹스·변주 규칙·sns_queue 출력 스키마만 압축해서 넣고, 긴 예시는 별 문서로 유지하면 좋습니다. [apaya](https://apaya.com/blog/ai-content-calendar)
- SNS 직원 페이지는  
  - 상단: 프로필 + 이번 주 요약,  
  - 본문: 좌측 캘린더 / 우측 큐 리스트 + 상세 에디터,  
  - 보조: “직접 시키기” 패널 + 성과/인사이트 섹션  
  구성으로 잡으면, 실제로 **주간 배치 생성 → 변주 관리 → 예약·승인 → 성과 피드백 → 광고/비주얼 연동**까지 한 화면에서 돌아가는 “실전형 SNS 오피스”가 될 것 같습니다. [taskade](https://www.taskade.com/generate/social-media-calendar/multi-channel-social-calendar)
