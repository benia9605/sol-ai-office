## 1. 회사 브레인 입력 버그 + 직원 정비 (커밋 `4e15619`)

처음 요청한 4가지를 처리:

### (1) 회사 브레인 입력칸 닫힘 버그 — 수정
- **증상**: 회사 브레인에서 직접 타이핑하면 한 글자마다 입력칸이 닫히고 화면이 위로 튐 (복붙은 됨)
- **원인**: `Field` 컴포넌트가 `BrandView` **함수 안에** 정의돼 있어서, 키 입력 → `setForm` → 리렌더 → `Field`가 매번 "새 컴포넌트"가 됨 → React가 `<input>`을 언마운트/리마운트 → 포커스 유실
- **수정**: `Field`를 모듈 스코프(컴포넌트 밖)로 이동, `value`/`onChange`를 props로 전달
- 파일: `src/components/office/BrandView.tsx`

### (2) 매일 자동으로 하는 일 — SOP대로 정리 + 시간 제거 + 클릭 설명
- 9개 직원 SOP에서 추출해 기본 일과 라벨에서 **시간 표현 제거** (`매일 09:00 카테고리 트렌드 점검` → `카테고리 트렌드 점검`). 시간/주기는 일과 ✎에서 직접 설정
- `ROUTINE_GUIDE` 추가 — 일과를 누르면 "정확히 무슨 일인지" SOP 기반 설명이 펼쳐짐 (옛 라벨도 `routineGuide()` 정규화로 매칭)
- 파일: `src/data/staffCatalog.ts`, `src/components/office/StaffView.tsx`

### (3) 직원마다 상단 온보딩 가이드
- 직원 상세 + 채용 모달 상단에 "💡 이 직원이 무슨 일을 하는지 + 사용법" 가이드 카드
- 파일: `src/data/staffCatalog.ts`(`guide` 필드), `StaffView.tsx`, `HireStaffModal.tsx`, `src/types.ts`

### (4) AI 셋팅 디테일 + 숨은 버그 수정
- 채용 시 일과별 설명 표시, "주기·시간은 채용 후 설정" 안내
- **버그 수정**: 모델 변경(🧠)이 `updateStaff`가 `model`을 안 받아 조용히 무시되던 것 → `model` 저장 추가

## 2. '일정 비서'(scheduler) AI 직원 신설 (커밋 `067da40`)

"콘텐츠 캘린더 말고, 비서처럼 내 일정 짜주는 게 누구냐?" → 없어서 신설.

- **방식**: 제안 → 사장 승인 → 캘린더 등록 (기존 `staff_output_actions` type=schedule → `approveAction` → `addSchedule` 파이프라인 재사용)
- **직접 시키기 3모드**: `하루 일정 짜기` / `주간 일정 짜기` / `일정 정리·조정` (모드별 입력칸·전용 지시문)
- **기본 자동 루틴**: `오늘 일정 정리`, `주간 일정 계획`
- **실제 데이터 기반**: 실행 시 기존 일정(개인+오피스)·미완료 할일·오늘 날짜를 AI 프롬프트에 주입 → 충돌 피해서 배치
- **전용 출력 카드**: `schedule_plan` — 타임블록 타임라인 + 미배치 + 경고 + 팁
- 파일: `staffCatalog.ts`, `staffInputForms.ts`, `staffRun.service.ts`, `StaffOutputView.tsx`, `StaffView.tsx`, `types.ts`(OutputKind `schedule_plan`), `supabase/functions/office-staff-run/index.ts`(자동실행 동기화)

## 3. 일정 "플랜" 스키마 + 오피스 일정에 개인 캘린더 이식 (커밋 `b439333`)

레퍼런스 `docs/guides/시목-오픈-스케줄러.html` (D-day + 주차별 체크리스트 + 마일스톤 + 카테고리) 기준.

### (1) SQL — `supabase/migrations/021_schedule_plans.sql`
- **`schedules` 확장**: `completed`(체크)·`completed_at`·`is_milestone`(절대 마감)·`plan_id`·`phase`(주차)·`sort_order`·`generated_by`(manual/ai)
- **`schedule_plans` 신설**: `name`·`emoji`·`goal`(예: 방문 30명+)·`target_date`(D-day)·`start_date`·`phases`(주차 JSONB)·`categories`(JSONB)·`status`·`generated_by`
- 매핑: HTML 날짜별 task = `schedules`행 / WEEKINFO = `phases` / MILESTONES = `is_milestone` / 상단 D-day·진행률 = `schedule_plans` + 완료 집계
- 동기화: `types.ts`, `schedules.service.ts`, `schedulePlans.service.ts`(신설), `mockSupabase.ts`, `docs/DATA_SCHEMA.md`

### (2) 개인 일정 UI에 완료 체크박스 추가
- 일정 행마다 원형 체크박스 → 완료 시 취소선/페이드, **마일스톤 🚩**·**AI 생성 🤖** 배지
- `useSchedules`: `workspaceId` 파라미터 + 완료/플랜 필드 흐름 + `toggleComplete`
- 파일: `src/hooks/useSchedules.ts`, `src/pages/SchedulesPage.modern.tsx`

### (3) 오피스 일정 = 개인 캘린더 이식
- 오피스의 단순 리스트 `ScheduleView` 폐기 → **개인 캘린더 `SchedulesPageModern`을 `embedded` + `workspaceId` 스코프로** 렌더
- 이제 오피스 일정에서도 미니캘린더·Featured·검색/필터·월별 그룹·체크박스·ICS·상세팝업·AI 배지 전부 동작
