# KPI 관리 + 할일-목표 연결 구현

## 현재 상황
- Part 1 (설정 메뉴 + 프로젝트 & 목표) 완료
- projects, goals, milestones 테이블 있음
- 할일(tasks)은 프로젝트에만 연결되어 있음

---

## 구조

```
프로젝트 (projects)
└── 목표 (goals)
    ├── KPI (kpis) - 수치 추적
    ├── 마일스톤 (milestones) - 중간 단계 체크
    └── 할일 (tasks) - 실제 작업 (goal_id로 연결)
```

### 예시

```
🔮 운명랩 (프로젝트)
└── 🎯 와디즈 펀딩 1,000만원 (목표)
    ├── 📊 KPI
    │   ├── 펀딩 금액: 350만원 / 1,000만원 (35%)
    │   └── 서포터 수: 42명 / 100명 (42%)
    ├── ✅ 마일스톤
    │   ├── ✅ 프롬프트 완료
    │   ├── ✅ 페이지 작성
    │   ├── 🔄 심사 통과
    │   └── ⬜ 펀딩 오픈
    └── 📋 할일
        ├── □ 썸네일 제작
        └── □ 상세 이미지 제작

✨ 쏠닝포인트 (프로젝트)
└── 🎯 인스타 팔로워 1,000명 (목표)
    ├── 📊 KPI
    │   └── 팔로워 수: 420명 / 1,000명 (42%)
    └── 📋 할일
        ├── □ 릴스 3개 제작
        └── □ 스토리 매일 올리기
```

## DB 변경

### 1. kpis 테이블 (새로 생성)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| goal_id | UUID | 목표 FK |
| name | TEXT | KPI 이름 ("펀딩 금액") |
| current_value | NUMERIC | 현재값 (350) |
| target_value | NUMERIC | 목표값 (1000) |
| start_value | NUMERIC | 시작값 (0) |
| unit | TEXT | 단위 ("만원", "명", "%") |
| created_at | TIMESTAMP | 생성일 |
| updated_at | TIMESTAMP | 수정일 |

### 2. kpi_logs 테이블 (새로 생성)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| kpi_id | UUID | KPI FK |
| value | NUMERIC | 기록 값 |
| date | DATE | 기록 날짜 |
| note | TEXT | 메모 |
| created_at | TIMESTAMP | 생성일 |

### 3. tasks 테이블 변경

| 추가 필드 | 타입 | 설명 |
|-----------|------|------|
| goal_id | UUID | 목표 FK (선택) |

## 목표 달성률 계산 로직

### KPI가 있는 경우
```
KPI 평균 달성률 표시
예: (35% + 42%) / 2 = 38.5%
```

### KPI가 없는 경우 (업무형)
```
연결된 할일 기준
완료된 할일 / 전체 할일
예: 3개 완료 / 5개 전체 = 60%
```

### 마일스톤만 있는 경우
```
완료된 마일스톤 / 전체 마일스톤
예: 2개 완료 / 4개 전체 = 50%
```

## 구현 순서

1. DB 테이블 추가 (kpis, kpi_logs, tasks에 goal_id)
2. 목표 상세 페이지에 KPI 섹션 추가
3. 목표 상세 페이지에 연결된 할일 표시
4. 할일 추가 모달에서 "상위 목표" 선택으로 변경
5. KPI 추가/편집/기록 기능
6. 목표 달성률 자동 계산
